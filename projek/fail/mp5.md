# Mini Projek: Latihan 5

## Setiap Ahli Edit Fail Sendiri

## Objektif Latihan

Setiap ahli kumpulan dapat menambah satu fail ringkas dalam repositori projek kumpulan, menulis maklumat peranan masing-masing dan melakukan commit supaya sumbangan setiap ahli dapat direkodkan.

## Situasi Latihan

Kumpulan telah menyediakan repositori, collaborator, Issues dan GitHub Projects Board. Dalam latihan ini, setiap ahli perlu membuat sumbangan kecil secara individu. Setiap ahli akan mencipta satu fail Markdown dalam folder `ahli` menggunakan nama atau GitHub ID masing-masing.

## Langkah 1: Buka Repositori Projek Kumpulan

1. Setiap ahli log masuk ke akaun GitHub masing-masing.
2. Buka repositori projek kumpulan.
3. Pastikan ahli mempunyai akses sebagai collaborator.
4. Semak tab `Issues` untuk melihat tugasan sendiri.
5. Semak GitHub Projects Board untuk melihat status tugasan.

## Langkah 2: Buka Issue Yang Diberikan

1. Buka tab `Issues`.
2. Cari Issue yang telah di-assign kepada diri sendiri.
3. Baca tajuk Issue.
4. Baca penerangan tugasan.
5. Pastikan tugasan yang akan dibuat sepadan dengan arahan dalam Issue.

## Langkah 3: Gerakkan Tugasan Ke In Progress

1. Buka GitHub Projects Board.
2. Cari kad tugasan sendiri.
3. Gerakkan kad daripada `To Do` ke `In Progress`.
4. Maklumkan kepada ahli kumpulan bahawa tugasan sedang dibuat.
5. Pastikan hanya kad tugasan sendiri yang digerakkan.

## Langkah 4: Cipta Folder ahli Jika Belum Ada

1. Buka tab `Code`.
2. Semak sama ada folder `ahli` sudah wujud.
3. Jika folder belum wujud, ahli pertama boleh menciptanya semasa menambah fail.
4. Folder ini akan menyimpan fail ringkas setiap ahli kumpulan.
5. Semua ahli perlu menggunakan folder yang sama.

Contoh struktur folder:

```text
nama-repositori/
└── ahli/
```

## Langkah 5: Tambah Fail Untuk Diri Sendiri

1. Klik `Add file`.
2. Pilih `Create new file`.
3. Pada ruangan nama fail, taip laluan fail dalam folder `ahli`.
4. Gunakan nama fail berdasarkan nama atau GitHub ID.

Contoh:

```text
ahli/ali.md
ahli/siti.md
ahli/ahmad-dev.md
ahli/nuraina.md
```

5. Elakkan ruang kosong dalam nama fail.
6. Gunakan huruf kecil jika boleh.

## Langkah 6: Tulis Maklumat Ahli

1. Dalam fail Markdown yang dicipta, tulis maklumat ringkas tentang diri.
2. Masukkan nama.
3. Masukkan GitHub ID.
4. Masukkan peranan dalam projek.
5. Masukkan tugasan yang dibuat.
6. Masukkan catatan ringkas tentang sumbangan.

Contoh kandungan fail:

```markdown
# Maklumat Ahli

- Nama: Ali bin Ahmad
- GitHub ID: ali-dev
- Peranan: Penulis README
- Tugasan: Mengemas kini dokumentasi projek

## Catatan Sumbangan

Saya bertanggungjawab menulis ringkasan projek, senarai ahli kumpulan dan status projek dalam README.
```

## Langkah 7: Tulis Mesej Commit

1. Scroll ke bahagian bawah halaman.
2. Cari bahagian `Commit changes`.
3. Tulis mesej commit yang jelas.
4. Mesej commit perlu menerangkan fail yang ditambah.

Contoh mesej commit:

```text
Tambah fail maklumat ahli Ali
```

Contoh lain:

```text
Tambah profil ahli kumpulan
```

## Langkah 8: Commit Perubahan

1. Semak semula nama fail.
2. Semak kandungan fail.
3. Pastikan mesej commit telah ditulis.
4. Klik `Commit changes`.
5. Tunggu sehingga GitHub menyimpan perubahan.

## Langkah 9: Semak Fail Ahli Dalam Repositori

1. Kembali ke tab `Code`.
2. Buka folder `ahli`.
3. Pastikan fail sendiri telah wujud.
4. Buka fail tersebut dan semak kandungan.
5. Jika ada kesilapan, klik edit dan commit perubahan baharu.

## Langkah 10: Gerakkan Tugasan Ke Review

1. Buka GitHub Projects Board.
2. Cari kad tugasan sendiri.
3. Gerakkan kad daripada `In Progress` ke `Review`.
4. Minta ahli lain menyemak fail yang telah ditambah.
5. Tunggu maklum balas daripada ahli kumpulan.

## Masalah Biasa dan Cara Mengatasi

| Masalah | Cadangan Penyelesaian |
|---|---|
| Tidak boleh tambah fail | Pastikan ahli telah menerima jemputan collaborator. |
| Folder `ahli` tidak wujud | Cipta folder semasa menambah fail menggunakan format `ahli/nama-fail.md`. |
| Nama fail tidak kemas | Gunakan huruf kecil dan elakkan ruang kosong. |
| Commit gagal | Semak mesej commit dan pastikan kandungan fail telah ditulis. |
| Fail tersimpan di lokasi salah | Pindahkan atau cipta semula fail dalam folder `ahli`. |
