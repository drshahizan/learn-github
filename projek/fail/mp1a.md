# Mini Project: Exercise 1

## Create a Group Project Repository

## Learning Objective

The group leader will be able to create one group project repository that will be used by all members throughout the mini project activity. This repository will become the group's work centre for storing files, writing the README, managing Issues, using Projects and carrying out collaboration.

## Exercise Scenario

Each group has 4 members. One member needs to act as the repository leader. This leader is responsible for creating the group project repository and preparing the initial settings so that other members can join the project in the next exercise.

## Step 1: Sign In to a GitHub Account

1. The group leader opens `https://github.com`.
2. The group leader signs in using their own GitHub account.
3. Make sure the account being used is the correct account.
4. Click the profile icon at the top right to check the GitHub username.
5. If using a lab computer, make sure no one else's account is signed in.

## Step 2: Open the Create New Repository Page

1. Click the `+` icon at the top right of GitHub.
2. Select the menu to create a new repository.
3. Make sure the create repository page is displayed.
4. Check the `Owner` section.
5. Make sure the `Owner` is the group leader's account.

## Step 3: Set the Repository Name

1. Find the repository name field.
2. Enter the group project repository name.
3. Use a name that is short, clear and professional.
4. Avoid names such as `test`, `project`, `exercise` or `abc`.
5. Use a name that reflects the group's mini project.

Example repository names:

```text
group-1-mini-project
mad-club
group-a-portfolio
club-activity-system
mini-project-github-team
```

## Step 4: Write the Repository Description

1. Find the `Description` field.
2. Write one short sentence that describes the group project.
3. The description should answer the question: what is this repository about?
4. Make sure the sentence is easy for group members and the facilitator to understand.
5. Do not leave the description field empty.

Example description:

```text
Mini project repository for group GitHub collaboration practice.
```

Another example:

```text
Group mini project for building a README, Issues, Projects and Pull Requests.
```

## Step 5: Choose Public or Private

1. Choose `Public` if the project is suitable to be viewed by the facilitator, friends and other participants.
2. Choose `Private` if the project contains sensitive information or is for closed training only.
3. For the workshop mini project, choose `Public` if the facilitator wants to review it easily.
4. If the group is unsure, ask the facilitator before choosing.
5. Remember that a public repository can be viewed by outsiders.

## Step 6: Enable README

1. Find the `Add a README file` option.
2. Tick that option.
3. The README will become the main document of the group project.
4. This file will be used in the next exercise to write project information.
5. Make sure this option is selected before the repository is created.

## Step 7: Additional Settings

1. The `.gitignore` section can be left empty for this exercise.
2. The `license` section can also be left empty for now.
3. The focus of this exercise is to create a group repository that can be used for collaboration.
4. Additional settings can be updated later if needed.
5. Do not change settings that you do not understand without instructions from the facilitator.

## Step 8: Create the Repository

1. Check the repository name.
2. Check the description.
3. Check the public or private option.
4. Make sure `Add a README file` has been selected.
5. Click the button to create the repository.
6. Wait until the new repository page is displayed.

## Step 9: Check the Group Project Repository

1. Make sure the repository name is displayed correctly.
2. Make sure the `README.md` file exists.
3. Check the main tabs such as `Code`, `Issues`, `Pull requests`, `Projects` and `Settings`.
4. Make sure the group leader can open the `README.md` file.
5. If the repository was created incorrectly, inform the facilitator before continuing.

## Step 10: Copy the Repository Link

1. Copy the repository link from the address bar.
2. The link usually looks like this:

```text
https://github.com/username/repository-name
```

3. Share the link with group members.
4. Save this link because it will be used in the add collaborator exercise.
5. Make sure all group members can open the link.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| Repository name has already been used | Choose another more specific name, for example by adding the group number. |
| Forgot to enable README | Create the `README.md` file manually after the repository is ready. |
| Accidentally chose public or private | Open `Settings` and change the visibility if needed. |
| Group members cannot open the link | Check the repository visibility again and make sure the link is correct. |
| Leader is using the wrong account | Sign out and sign in again using the correct leader's GitHub account. |
