# Mini Project: Exercise 3

## Create Issues for Task Distribution

## Learning Objective

Participants will be able to create Issues in the group project repository to distribute tasks to each member. Each group member should have at least one Issue that is clear, understandable and trackable throughout the mini project.

## Exercise Scenario

The group project repository has been created and all members have been added as collaborators. In this exercise, the group will use Issues as the official task list. Each Issue represents one task that needs to be completed by a group member.

## Step 1: Open the Group Project Repository

1. The group leader or task manager signs in to GitHub.
2. Open the group project repository.
3. Make sure the opened repository is the correct group project repository.
4. Make sure all members have become collaborators.
5. If any member is not yet a collaborator, complete Exercise 2 first.

## Step 2: Open the Issues Tab

1. On the repository page, click the `Issues` tab.
2. Make sure the Issues page is displayed.
3. If there is no Issue yet, the list will appear empty.
4. Click the `New issue` button.
5. GitHub will display the form to create a new Issue.

## Step 3: Set a Task for Each Member

1. The group needs to prepare at least 4 Issues.
2. Each group member should receive one task.
3. The task should be small, clear and possible to complete within the mini project time.
4. Avoid tasks that are too large or too general.
5. Make sure the task is related to the group project outcome.

Example task distribution:

| Member | Task |
|---|---|
| Member 1 | Update the project README |
| Member 2 | Add a member profile file |
| Member 3 | Add the project folder structure |
| Member 4 | Check links, spelling and Markdown format |

## Step 4: Write a Clear Issue Title

1. In the Issue title field, write a task title that is easy to understand.
2. The title should show the work that needs to be done.
3. Avoid titles that are too short, such as `do work`, `update` or `fix`.
4. Use clear action words.
5. Make sure the assigned member understands the meaning of the title.

Example Issue titles:

```text
Update the group project README
Add a profile file for a group member
Add an images folder for project materials
Check the README Markdown format and links
```

## Step 5: Write the Issue Description

1. In the description field, write clearer task information.
2. Explain what needs to be done.
3. State the related file or folder if applicable.
4. State the expected outcome after the task is completed.
5. Use a checklist so the task is easy to monitor.

Example Issue description:

```markdown
This task is to update the group project README.

Task list:

- Add the project name
- Add the project summary
- Add the group member list
- Add the project status

Expected outcome:

The project README contains complete basic information and is easy to read.
```

## Step 6: Assign the Issue to a Member

1. On the right side of the Issue, find the `Assignees` option.
2. Click `Assignees`.
3. Select the group member responsible for the task.
4. Make sure only the suitable member is selected.
5. If the member's name does not appear, check the collaborator status again in Exercise 2.

## Step 7: Add Labels If Time Allows

1. On the right side of the Issue, find the `Labels` option.
2. Select a suitable label if available.
3. Examples of labels that can be used:

```text
documentation
enhancement
bug
question
```

4. If there is no suitable label, the group may leave it without a label.
5. The main focus of this exercise is to create Issues and assign tasks.

## Step 8: Create the Issue

1. Check the Issue title.
2. Check the Issue description.
3. Check the assignee.
4. Click `Create`.
5. The Issue will be displayed in the Issues list.

## Step 9: Repeat for All Members

1. Repeat the Issue creation process until all members have tasks.
2. Make sure each Issue has a clear title.
3. Make sure each Issue has an understandable description.
4. Make sure each Issue has been assigned to the correct member.
5. Make sure at least 4 Issues have been created for a group of 4 members.

## Step 10: Check the Issues List

1. Return to the `Issues` tab.
2. Check all Issues that have been created.
3. Make sure each member has one Issue.
4. Make sure tasks do not overlap unnecessarily.
5. If there is a mistake, open the Issue and update the information.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| Member's name does not appear in Assignees | Make sure the member has accepted the collaborator invitation. |
| Issue is too general | Break the task into smaller and clearer work. |
| A member receives too many tasks | Redistribute the tasks so they are more balanced. |
| Issue description is unclear | Add a task list and expected outcome. |
| Issue is assigned to the wrong member | Reopen the Issue and change the assignee to the correct member. |
