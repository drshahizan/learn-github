# Mini Project: Exercise 4

## Create a GitHub Projects Board

## Learning Objective

Participants will be able to create a GitHub Projects Board to organise group tasks, connect the Issues that have been created and visually monitor each member's work status.

## Exercise Scenario

The group has created Issues for task distribution in Exercise 3. In this exercise, all Issues will be added to a GitHub Projects Board so that task status can be monitored through the `To Do`, `In Progress`, `Review` and `Done` columns.

## Step 1: Open the Group Project Repository

1. The group leader or task manager signs in to GitHub.
2. Open the group project repository.
3. Make sure the opened repository is the correct group project repository.
4. Make sure Issues for group members have been created.
5. If Issues do not exist yet, complete Mini Project Exercise 3 first.

## Step 2: Open the Projects Tab

1. On the repository page, click the `Projects` tab.
2. If the `Projects` tab is not visible, check the repository menu or ask the facilitator.
3. Click the button to create a new project.
4. Choose a suitable view, such as board or table.
5. For this exercise, use the board view because it is easier to see work status.

## Step 3: Name the Project Board

1. Enter a clear project board name.
2. Use a name that describes the group project.
3. Avoid names that are too general, such as `test` or `project`.

Example project board names:

```text
Group 1 Mini Project
Mad_club Task Board
GitHub Project Collaboration
```

4. Save the project board after entering the name.
5. Make sure the project board can be opened by group members.

## Step 4: Prepare Task Columns

1. Prepare the main columns for task status.
2. Use the following four columns:

```text
To Do
In Progress
Review
Done
```

3. `To Do` is used for tasks that have not started.
4. `In Progress` is used for tasks currently being worked on.
5. `Review` is used for tasks that need to be checked.
6. `Done` is used for completed tasks.

## Step 5: Add Issues to the Board

1. Open the project board that has been created.
2. Add a new item to the board.
3. Select the Issues created in Exercise 3.
4. Add all group Issues into the `To Do` column.
5. Make sure each member has at least one task in the board.

## Step 6: Check Each Member's Task

1. Open each task card on the board.
2. Check the task title.
3. Check the assignee for the task.
4. Make sure the task has been assigned to the correct member.
5. If a task is assigned incorrectly, open the related Issue and update the assignee.

## Step 7: Move Tasks According to Status

1. Tasks that have not started remain in the `To Do` column.
2. When a member starts working, move the task to `In Progress`.
3. When the task is ready to be checked, move it to `Review`.
4. After the task is accepted or merged, move it to `Done`.
5. Make sure all members update their own task status.

## Step 8: Use the Board During Group Discussion

1. The group leader opens the project board during a short discussion.
2. Each member explains the status of their task.
3. Identify tasks that have not started.
4. Identify tasks that need help.
5. Update the task card positions after the discussion.

## Step 9: Connect the Board with the Next Work

1. Use the board as a reference when members start editing files.
2. Make sure tasks being worked on are in `In Progress`.
3. After a member commits or opens a Pull Request, move the task to `Review`.
4. After the task is completed, move it to `Done`.
5. Use this board throughout the mini project until the final presentation.

## Step 10: Check the Project Board

1. Make sure all Issues are in the project board.
2. Make sure each member has a task.
3. Make sure the `To Do`, `In Progress`, `Review` and `Done` columns exist.
4. Make sure at least one task has been moved according to work status.
5. Share the board view with the facilitator if requested.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| Projects tab is not visible | Check the repository menu or ask the facilitator for help. |
| Issues do not appear in the item options | Make sure the Issues have been created in the same repository. |
| Task has no assignee | Open the related Issue and assign it to the correct member. |
| Board columns are not suitable | Rename the columns to `To Do`, `In Progress`, `Review` and `Done`. |
| Members do not know task status | Use the board during a short group discussion. |
