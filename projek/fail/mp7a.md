# Mini Project: Exercise 7

## Update the Project README

## Learning Objective

Participants will be able to update the main `README.md` file in the group project repository so that the project information, group members, roles, member file links and project status are displayed clearly.

## Exercise Scenario

The group has created a repository, added collaborators, created Issues, used a Projects Board, added member files and carried out Pull Requests. In this final exercise, the group needs to update the main README so that the project repository looks complete and is easy for the facilitator to evaluate.

## Step 1: Open the Group Project Repository

1. The group leader or README writer signs in to GitHub.
2. Open the group project repository.
3. Make sure the opened repository is the correct group project repository.
4. Check that member files exist in the `ahli` folder.
5. Check that the main tasks have been updated in the GitHub Projects Board.

## Step 2: Open the README.md File

1. In the `Code` tab, find the `README.md` file.
2. Click the `README.md` file.
3. Click the pencil icon or the `Edit` button.
4. GitHub will open the README editor.
5. Make sure the README writer does not delete important content without group agreement.

## Step 3: Write the Project Name

1. Place the project title at the top of the README.
2. Use the `#` symbol for the main heading.
3. The project name should be the same as or close to the repository name.
4. Make sure the project title is short and easy to understand.

Example:

```markdown
# Group 1 GitHub Mini Project
```

Another example:

```markdown
# Mad_club: GitHub Collaboration Mini Project
```

## Step 4: Write the Project Summary

1. Add a `Project Summary` section.
2. Explain the purpose of the project in one short paragraph.
3. State that this project is used for GitHub collaboration practice.
4. Make sure the sentences are easy for outside readers to understand.

Example:

```markdown
## Project Summary

This project was developed as a mini exercise to understand team collaboration using GitHub. The group uses a repository, Issues, GitHub Projects, Pull Requests and README documentation to organise shared work.
```

## Step 5: List Group Members and Roles

1. Add a `Group Members` section.
2. Use a Markdown table.
3. Enter each member's name.
4. Enter each member's GitHub ID.
5. Enter each member's role in the project.

Example:

```markdown
## Group Members

| Name | GitHub ID | Role |
|---|---|---|
| Member Name 1 | member1 | Repository leader |
| Member Name 2 | member2 | README writer |
| Member Name 3 | member3 | Issues and Projects manager |
| Member Name 4 | member4 | Pull Request reviewer |
```

## Step 6: Add Member File Links

1. Open the `ahli` folder.
2. Copy each member's file name.
3. In the README, add a `Member File Links` section.
4. Add links to member files using Markdown format.
5. Make sure all links can be clicked.

Example:

```markdown
## Member File Links

- [Ali](ahli/ali.md)
- [Siti](ahli/siti.md)
- [Ahmad](ahli/ahmad.md)
- [Nuraina](ahli/nuraina.md)
```

## Step 7: Add a Collaboration Summary

1. Add a `Collaboration Summary` section.
2. State the GitHub activities used by the group.
3. Include Issues, Projects Board, Pull Requests and reviews.
4. Use bullet points so it is easy to read.

Example:

```markdown
## Collaboration Summary

- The group project repository has been created.
- All members have been added as collaborators.
- Issues were used for task distribution.
- GitHub Projects was used to monitor work status.
- Pull Requests were used to review changes.
```

## Step 8: State the Project Status

1. Add a `Project Status` section.
2. State the current project status.
3. Use a status that is easy to understand.
4. The status can be updated after the presentation if needed.

Example:

```markdown
## Project Status

Status: Ready for facilitator review
```

Other status options:

```markdown
Status: Under group review
Status: Ready for mini project presentation
```

## Step 9: Preview the README

1. Click the `Preview` tab.
2. Check the project title display.
3. Check the group member table.
4. Check the member file links.
5. Check the collaboration summary list.
6. Fix the Markdown format if any section is not neat.

## Step 10: Commit README Changes

1. Scroll to the bottom of the editor.
2. Find the `Commit changes` section.
3. Write a clear commit message.

Example commit message:

```text
Update group project README
```

Another example:

```text
Add group members and project status
```

4. Click `Commit changes`.
5. Wait until the changes are saved.

## Step 11: Check the Final Repository Display

1. Return to the main repository page.
2. Check the README display below the file list.
3. Make sure all main sections are displayed.
4. Click the member file links to make sure they work.
5. Show the final display to the facilitator if requested.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| README table is not neat | Check the `|` symbols and the table separator row. |
| Member file links do not work | Check the folder name, file name and uppercase or lowercase letters. |
| README is too short | Add the project summary, group members and project status. |
| Forgot to commit changes | Scroll down and click `Commit changes`. |
| A member has no file | Ask the member to complete Mini Project Exercise 5. |
