# Mini Project: Exercise 6

## Pull Request and Team Review

## Learning Objective

Participants will be able to open a Pull Request for the changes made, conduct a team review, give short comments and merge suitable changes into the group project.

## Exercise Scenario

Each member has added their own file and moved the task to the `Review` status. In this exercise, the changes will be checked by another member through a Pull Request. This activity practises a collaboration process commonly used in real projects.

## Step 1: Make Sure the Task Is in Review

1. Open the group project repository.
2. Open the GitHub Projects Board.
3. Check your own task card.
4. Make sure the task card is in the `Review` column.
5. If it is still in `In Progress`, move it to `Review` before opening a Pull Request.

## Step 2: Open the Pull Requests Tab

1. On the repository page, click the `Pull requests` tab.
2. Click the `New pull request` button.
3. Make sure the change comparison page is displayed.
4. Check the source branch and target branch.
5. If using your own branch, make sure the source branch is your task branch.

## Step 3: Choose the Correct Branch

1. Select the source branch that contains your changes.
2. Select the target branch set by the group or facilitator.
3. Usually, the target branch is `main`.
4. Check the list of changed files.
5. Make sure only your task files are involved.

Example source branches:

```text
feature/ali-profile
siti-member-exercise
update-readme-ahmad
```

## Step 4: Check Changes Before the Pull Request

1. View the file changes section.
2. Check the content that was added or changed.
3. Make sure there is no sensitive information.
4. Make sure the file is in the correct folder.
5. If there is a mistake, return to the original file, fix it and commit again before continuing.

## Step 5: Write the Pull Request Title

1. Write a clear Pull Request title.
2. The title should explain the change made.
3. Avoid titles that are too general, such as `update`, `edit` or `test`.
4. Use the member name or task if suitable.

Example Pull Request titles:

```text
Add Ali member information file
Update the group project README
Add member folder structure
```

## Step 6: Write the Pull Request Description

1. Explain the changes that have been made.
2. State the files involved.
3. State the related Issue if applicable.
4. Write a short and easy-to-understand description.

Example description:

```markdown
This Pull Request adds a member information file in the `ahli` folder.

Changes made:

- Add `ahli/ali.md`
- Add name, GitHub ID and role
- Update task status for review
```

## Step 7: Submit the Pull Request

1. Check the Pull Request title.
2. Check the Pull Request description.
3. Check the list of changed files.
4. Click the button to create the Pull Request.
5. Copy the Pull Request link if requested by the facilitator.

## Step 8: Another Member Reviews It

1. Another member opens the Pull Request.
2. Read the Pull Request title and description.
3. Check the changed files.
4. Leave a short comment if there is a suggested correction.
5. If the changes are good, write an approval comment.

Example review comment:

```text
The file is in the correct folder and the member information is complete.
```

Example correction comment:

```text
Please correct the GitHub ID spelling before merging.
```

## Step 9: Fix Changes If Needed

1. If the reviewer requests corrections, the member who created the Pull Request needs to edit the file again.
2. Commit the correction.
3. Push the changes if using GitHub Desktop.
4. The Pull Request will be updated automatically.
5. Ask the reviewer to check again.

## Step 10: Merge the Pull Request

1. After the review is complete, the group leader or an authorised member can merge the Pull Request.
2. Click the `Merge pull request` button.
3. Confirm the merge process.
4. Make sure the Pull Request status changes to merged.
5. Do not merge if there are still mistakes that have not been fixed.

## Step 11: Close the Related Issue

1. Open the Issue related to the Pull Request.
2. Check whether the task has been completed.
3. If the task is complete, close the Issue.
4. Write a short comment if needed.
5. Make sure group members know the task has been completed.

## Step 12: Move the Task to Done

1. Open the GitHub Projects Board.
2. Find the completed task card.
3. Move the card from `Review` to `Done`.
4. Check all other members' tasks.
5. Make sure the board status reflects the group's actual work.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| Pull Request cannot be opened | Make sure the changes were made in the correct branch. |
| Too many files changed | Check the files again and avoid unrelated changes. |
| Reviewer cannot see the changes | Make sure the branch has been pushed to GitHub. |
| Cannot merge | Check whether there are conflicts or unfinished reviews. |
| Issue is not closed | Close the Issue manually after the task is complete. |
