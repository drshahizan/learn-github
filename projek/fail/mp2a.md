# Mini Project: Exercise 2

## Add Members as Collaborators

## Learning Objective

The group leader will be able to add three group members as collaborators in the group project repository so that all members can work together, edit files, create Issues, update Projects and contribute to the mini project.

## Exercise Scenario

The group project repository has been created in Exercise 1. In this exercise, the group leader needs to invite three other members as collaborators. Each member must accept the invitation before they can contribute to the group repository.

## Step 1: The Leader Opens the Group Project Repository

1. The group leader signs in to their GitHub account.
2. Open the group project repository created in Exercise 1.
3. Make sure the opened repository is the correct group project repository.
4. Check the repository name at the top of the page.
5. Make sure the group leader has access to the `Settings` tab.

## Step 2: Open the Settings Tab

1. On the repository page, find the `Settings` tab.
2. Click the `Settings` tab.
3. If the `Settings` tab is not visible, make sure the leader is using the repository owner's account.
4. Wait until the repository settings page is displayed.
5. Do not change other settings without instructions from the facilitator.

## Step 3: Open the Collaborators Section

1. On the `Settings` page, find the `Collaborators` section.
2. If GitHub shows the `Collaborators and teams` menu, select that menu.
3. GitHub may ask for password confirmation.
4. Enter the password if requested.
5. Make sure the page for adding collaborators is displayed.

## Step 4: Get Group Members' GitHub IDs

1. Ask each group member to provide their GitHub ID.
2. A GitHub ID is the username shown in the GitHub profile link.
3. Example profile link:

```text
https://github.com/username
```

4. In the example, the GitHub ID is `username`.
5. Record all members' GitHub IDs so the invitations are not sent to the wrong accounts.

Example list of group GitHub IDs:

```text
group-leader
first-member
second-member
third-member
```

## Step 5: Invite the First Member

1. Click the button to add a collaborator.
2. Enter the first member's GitHub ID.
3. Select the correct account from the suggestions displayed.
4. Click the button to send the invitation.
5. Inform the first member to check the invitation in their GitHub account.

## Step 6: Invite the Second and Third Members

1. Repeat the same steps for the second member.
2. Repeat the same steps for the third member.
3. Make sure each GitHub ID is spelled correctly.
4. Do not invite accounts that are not related to the group.
5. Check the invitation list after all members have been invited.

## Step 7: Members Accept the Invitation

1. Each member signs in to their own GitHub account.
2. Check GitHub notifications.
3. Open the collaborator invitation from the group leader.
4. Click the button to accept the invitation.
5. After accepting the invitation, members can open the group project repository.

## Step 8: Check Member Access

1. The group leader returns to the `Collaborators` page.
2. Check the collaborator list.
3. Make sure all three members have accepted the invitation.
4. Ask each member to open the repository link.
5. Make sure each member can see the required tabs such as `Code`, `Issues` and `Projects`.

## Step 9: Test Basic Access

1. Each member opens the group project repository.
2. Each member checks whether they can open the `README.md` file.
3. Each member checks whether they can open the `Issues` tab.
4. One member tries to add a short comment to an Issue if an Issue is already available.
5. If there is no Issue yet, this activity can be done in the next exercise.

## Step 10: Record the Collaborator Status

1. The group leader records the names of members who have successfully become collaborators.
2. Record each member's GitHub ID.
3. Record members who have not yet accepted the invitation.
4. If any member has not accepted the invitation, ask them to check GitHub notifications or email.
5. Make sure all members are ready before moving to the Issues and Projects exercises.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| Settings tab is not visible | Make sure the leader is using the repository owner's account. |
| Member's GitHub ID cannot be found | Check the spelling of the GitHub ID and make sure the member's account exists. |
| Invitation is not received | Ask the member to check GitHub notifications and email. |
| Member is using the wrong account | Ask the member to sign out and sign in using the correct GitHub account. |
| Member cannot access the repository | Check the collaborator status and repository visibility again. |
