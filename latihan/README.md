# Latihan Bengkel Asas GitHub

Modul latihan ini disediakan untuk peserta **Bengkel Asas GitHub**. Fokus latihan ialah penggunaan GitHub secara praktikal melalui pelayar web, GitHub Desktop, Visual Studio Code dan Google Colab. Latihan ini menekankan pembinaan profil profesional, repositori projek, dokumentasi README, pengurusan tugasan, portfolio digital, keselamatan dan etika akademik.

## Objektif Latihan

Selepas melengkapkan latihan, peserta dapat:

1. Mencipta dan mengemas kini akaun GitHub.
2. Membina profil GitHub yang sesuai untuk portfolio digital.
3. Menghasilkan repositori projek pertama.
4. Menulis README yang lengkap dan kemas.
5. Menggunakan Markdown asas dalam fail README.
6. Memuat naik fail projek, imej dan pautan demo.
7. Mengurus tugasan menggunakan Issues dan GitHub Projects.
8. Menghubungkan GitHub dengan GitHub Desktop, Visual Studio Code dan Google Colab.
9. Mengamalkan keselamatan asas, etika akademik dan hak cipta.

## Hasil Akhir Peserta

Pada akhir bengkel, peserta dijangka mempunyai:

- Akaun GitHub yang aktif.
- Profil GitHub yang lebih profesional.
- Sekurang-kurangnya satu repositori projek.
- README projek yang mempunyai tajuk, ringkasan, ciri utama, teknologi, ahli kumpulan dan status projek.
- Tangkapan skrin atau pautan demo jika ada.
- Pautan profil atau repositori yang boleh dikongsi kepada fasilitator.

## Perincian Ringkas Latihan

### Latihan 1: Cipta Akaun GitHub

Peserta mencipta akaun GitHub menggunakan emel aktif, memilih nama pengguna yang sesuai, menetapkan kata laluan, menyelesaikan pengesahan keselamatan dan menyalin pautan profil untuk kegunaan latihan seterusnya.

### Latihan 2: Follow Profil GitHub Dr Shahizan

Peserta membuka profil [github.com/drshahizan](https://github.com/drshahizan), klik `Follow`, menyemak status following dan meneroka sekurang-kurangnya satu repositori rujukan.

### Latihan 3: Profil GitHub

Peserta mengemas kini gambar profil, nama paparan, bio, lokasi dan pautan tambahan supaya profil GitHub lebih sesuai untuk tujuan akademik dan portfolio digital.

### Latihan 4: Hasilkan Repositori Projek Pertama

Peserta mencipta repositori baharu, menulis description, memilih public atau private, mengaktifkan `README.md` dan menambah maklumat awal dalam README.

### Latihan 5: Repositori Mad_club

Peserta menulis README projek untuk `Mad_club` yang mengandungi tajuk projek, ringkasan projek, ciri utama, teknologi digunakan, ahli kumpulan dan status projek.

### Latihan 6: Penggunaan Markdown Dalam Fail README

Peserta menggunakan Markdown asas seperti heading, bullet list, numbered list, pautan, imej, jadual dan kod ringkas untuk menjadikan README lebih tersusun.

### Latihan 7: Muat Naik Fail Projek

Peserta memuat naik fail projek melalui GitHub web, menyusun fail dalam folder seperti `css`, `js`, `images` atau `assets`, menulis mesej commit dan menyemak perubahan.

### Latihan 8: Imej Dalam README

Peserta menyediakan folder `images` atau `assets`, memuat naik tangkapan skrin projek, memaparkan imej dalam README dan menambah pautan demo jika ada.

### Latihan 9: Gunakan Issues Untuk Tugasan

Peserta mencipta Issue untuk merekod tugasan, soalan, masalah atau cadangan penambahbaikan projek.

### Latihan 10: Susun Tugasan Menggunakan GitHub Projects

Peserta membina Projects Board dengan lajur seperti `To Do`, `In Progress`, `Review` dan `Done`.

### Latihan 11: Kolaborasi Projek Berkumpulan

Peserta memahami peranan collaborator, komen dalam Issue dan konsep Pull Request sebagai ruang semakan kerja.

### Latihan 12: Repositori Public vs Private

Peserta memilih tetapan public atau private berdasarkan tujuan projek, sensitiviti data dan kesesuaian portfolio.

### Latihan 13: Semakan Keselamatan Repositori

Peserta menyemak repositori supaya tiada kata laluan, token, kunci API, data peribadi atau tugasan sensitif dimuat naik.

### Latihan 14: Terbitkan Laman Ringkas Dengan GitHub Pages

Peserta mengaktifkan GitHub Pages melalui tetapan repositori dan menambah pautan demo ke dalam README.

### Latihan 15: Pin Repositori Pada Profil

Peserta memilih repositori terbaik dan memaparkannya pada profil GitHub sebagai bahan portfolio.

### Latihan 16: Bina Portfolio Digital Mini

Peserta menggabungkan profil GitHub, repositori pilihan, README, imej dan pautan demo menjadi portfolio ringkas.

### Latihan 17: Pengenalan GitHub Desktop

Peserta memahami fungsi GitHub Desktop sebagai aplikasi visual untuk kerja lokal seperti clone, commit, push dan pull.

### Latihan 18: Pasang dan Log Masuk GitHub Desktop

Peserta memasang GitHub Desktop, log masuk menggunakan akaun GitHub dan menyemak akaun aktif.

### Latihan 19: Clone Repositori Ke Komputer

Peserta menyalin repositori daripada GitHub ke komputer menggunakan GitHub Desktop.

### Latihan 20: Buat Perubahan Fail Secara Lokal

Peserta membuka fail projek pada komputer, mengedit fail dan melihat perubahan yang dikesan dalam GitHub Desktop.

### Latihan 21: Commit dan Push Menggunakan GitHub Desktop

Peserta menulis commit summary, commit perubahan dan push perubahan ke GitHub.

### Latihan 22: Pull Perubahan Terkini

Peserta mengambil perubahan terkini daripada GitHub supaya fail lokal sentiasa selari dengan versi dalam talian.

### Latihan 23: Branch dan Pull Request Secara Visual

Peserta mencipta branch, membuat perubahan kecil, publish branch dan membuka Pull Request melalui GitHub web.

### Latihan 24: Pengenalan GitHub Dalam Visual Studio Code

Peserta memahami fungsi VS Code, Source Control dan hubungan VS Code dengan GitHub.

### Latihan 25: Hubungkan VS Code Dengan Akaun GitHub

Peserta log masuk ke GitHub melalui VS Code dan memastikan akaun aktif adalah betul.

### Latihan 26: Buka Repositori Dalam VS Code

Peserta membuka folder repositori dalam VS Code dan mengenal pasti fail projek.

### Latihan 27: Commit dan Sync Melalui VS Code

Peserta menggunakan Source Control untuk melihat perubahan, commit dan sync ke GitHub.

### Latihan 28: Pengenalan GitHub Dalam Google Colab

Peserta memahami fungsi Google Colab untuk notebook Python, data, AI dan analitik.

### Latihan 29: Buka Notebook GitHub Dalam Google Colab

Peserta membuka fail `.ipynb` daripada GitHub menggunakan Google Colab.

### Latihan 30: Simpan Notebook Google Colab Ke GitHub

Peserta menyimpan salinan notebook Colab ke repositori GitHub dengan mesej commit yang jelas.

### Latihan 31: Dokumentasi Projek Notebook

Peserta menulis README yang menerangkan objektif notebook, dataset, cara membuka notebook dan amaran berkaitan data sensitif.

### Latihan 32: Pengenalan GitHub Copilot Secara Bertanggungjawab

Peserta memahami Copilot sebagai pembantu AI dan menyemak cadangan sebelum digunakan.

### Latihan 33: Etika Akademik dan Hak Cipta

Peserta belajar memberi kredit, memahami lesen dan tidak mengaku kod orang lain sebagai hasil sendiri.

### Latihan 34: Senarai Semak Akhir Sebelum Dikongsi

Peserta menyemak repositori dari sudut visibility, README, data sensitif, tangkapan skrin dan pautan demo.

### Latihan 35: Pembentangan Ringkas Portfolio GitHub

Peserta membentangkan profil GitHub, repositori pilihan, README, imej projek dan pautan demo.

## Nota Keselamatan

Jangan muat naik perkara berikut ke dalam repositori:

- Kata laluan.
- Token.
- Kunci API.
- Fail konfigurasi sulit.
- Nombor kad pengenalan.
- Data peribadi.
- Tugasan sensitif yang belum dibenarkan untuk dikongsi.
