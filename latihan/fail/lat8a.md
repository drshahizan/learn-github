# Exercise 8: Images in README

## Learning Objective

Participants will be able to prepare an image folder in a repository, upload project screenshots, display images in the `README.md` file and add a demo link if available.

## Step 1: Prepare Project Screenshots

1. Open the project that you want to display.
2. Take a screenshot of the homepage, system display or project output.
3. Choose the clearest image that best represents the project.
4. Avoid images that are blurry, too small or unrelated.
5. Save the image in a suitable format such as `.png`, `.jpg` or `.jpeg`.

Examples of suitable file names:

```text
main-display.png
madclub-demo.png
activity-page.png
```

## Step 2: Name Image Files Neatly

1. Use short file names.
2. Use lowercase letters if possible.
3. Avoid spaces in file names.
4. Use hyphens to separate words.
5. Make sure the file name is easy to refer to in README.

Examples of good file names:

```text
main-display.png
activity-list.png
registration-form.png
```

Examples of less suitable file names:

```text
Screenshot 2026-05-27 at 10.30.11 PM.png
image final latest correct.png
new project picture copy.png
```

## Step 3: Prepare an images or assets Folder

1. Open the project repository in GitHub.
2. Click `Add file`.
3. Select `Create new file`.
4. In the file name field, type the folder name followed by `/`.

Example:

```text
images/main-display.png
assets/main-display.png
```

GitHub web does not create empty folders. A folder only exists when there is a file inside it.

## Step 4: Upload Screenshots to the Image Folder

1. Open the `images` or `assets` folder in the repository.
2. Click `Add file`.
3. Select `Upload files`.
4. Drag the image file into the upload area.
5. Make sure the image file has been uploaded successfully.
6. Scroll to the bottom and write a commit message.

Example commit message:

```text
Add project screenshot
```

7. Click `Commit changes`.

## Step 5: Copy the Image File Path

1. Open the uploaded image file.
2. Check the file location in the repository.
3. If the image is in the `images` folder, the path usually looks like this:

```text
images/main-display.png
```

4. If the image is in the `assets` folder, the path usually looks like this:

```text
assets/main-display.png
```

5. Copy the path to use in the README.

## Step 6: Display the Image in README

1. Open the `README.md` file.
2. Click the pencil icon or the `Edit` button.
3. Find a suitable section to place the image.
4. Add a new section such as `Project Display`.
5. Use the following Markdown format:

```markdown
## Project Display

![Mad_club main display](images/main-display.png)
```

If using the `assets` folder, use:

```markdown
## Project Display

![Mad_club main display](assets/main-display.png)
```

## Step 7: Preview the README

1. Click the `Preview` tab.
2. Check whether the image is displayed.
3. If the image does not appear, check the folder name.
4. Check the file name and uppercase or lowercase letters.
5. Check whether the Markdown format is correct.
6. Fix the image path if needed.

## Step 8: Add a Demo Link If Available

1. If the project has a demo, add a `Project Demo` section.
2. The demo may be GitHub Pages, a demonstration video, a project website, Google Colab or an approved APK file.
3. Use Markdown link format.

Example:

```markdown
## Project Demo

[Mad_club Demo](https://username.github.io/Mad_club/)
```

Example for video:

```markdown
## Project Demo

[Mad_club Demo Video](https://example.com/video-demo)
```

## Step 9: Commit README Changes

1. Scroll to the bottom of the README editor.
2. Find the `Commit changes` section.
3. Write a clear commit message.

Example commit message:

```text
Add image and demo link to README
```

4. Click `Commit changes`.
5. Wait until GitHub saves the changes.

## Step 10: Check the Final Display

1. Return to the main repository page.
2. Check the README display.
3. Make sure the image appears clearly.
4. Click the demo link if available.
5. Make sure the demo link works.
6. If there is a problem, edit the README and commit the new changes.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| Image does not appear in README | Check the image path, folder name and file name. |
| File name has spaces | Rename the file using hyphens, such as `main-display.png`. |
| Uppercase and lowercase letters do not match | Make sure the spelling in README is the same as the actual file name. |
| Image is too large | Reduce the image size before uploading. |
| Demo link does not work | Open the link in a new tab and check the URL again. |
