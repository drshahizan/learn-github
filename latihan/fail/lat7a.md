# Exercise 7: Upload Project Files

## Learning Objective

Participants will be able to upload project files into a repository through GitHub web, organise files in suitable folders, write clear commit messages and review the changes in the repository.

## Step 1: Open the Project Repository

1. Sign in to your GitHub account.
2. Open the project repository you want to update, for example `Mad_club`.
3. Make sure you are on the `Code` tab.
4. Check the list of files that already exist in the repository.
5. Make sure the repository opened is the correct repository.

## Step 2: Prepare Project Files on the Computer

1. Prepare the files you want to upload.
2. Make sure the file names are short and easy to understand.
3. Avoid file names that are too long.
4. Avoid spaces in file names.
5. Use hyphens, such as `main-display.png`.

Example project files:

```text
index.html
style.css
script.js
README.md
logo.png
main-display.png
```

## Step 3: Plan the Folder Structure

1. Decide suitable folders for the project files.
2. Main files such as `index.html` can be placed at the main level of the repository.
3. CSS files can be placed in the `css` folder.
4. JavaScript files can be placed in the `js` folder.
5. Images can be placed in the `images` or `assets` folder.

## Step 4: Select the Upload Files Function

1. On the repository page, click the `Add file` button.
2. Select `Upload files`.
3. GitHub will open the page for uploading files.
4. Make sure the page shows the correct repository name.
5. If the wrong repository is selected, go back before uploading the files.

## Step 5: Upload Files

1. Drag the files from your computer into the upload area.
2. Participants may also click `choose your files`.
3. Select the files to upload.
4. Wait until all files have finished uploading.
5. Make sure no file failed to upload.

## Step 6: Organise Files in Folders

1. If you want to create a folder through GitHub web, click `Add file` and select `Create new file`.
2. In the file name field, type the folder name followed by `/`.
3. Example:

```text
images/main-display.png
```

4. For a new file, GitHub will create the `images` folder.
5. If files have already been uploaded without a folder, participants may leave them for now or reorganise them in an advanced exercise.

## Step 7: Write a Clear Commit Message

1. Scroll to the bottom of the upload page.
2. Find the `Commit changes` section.
3. Write a short commit message that explains the change.
4. Avoid commit messages that are too general, such as `update`, `test` or `new file`.

Examples of good commit messages:

```text
Add basic Mad_club project files
Add main project screenshot
Add initial HTML and CSS files
```

## Step 8: Commit the Changes

1. Review the list of files that will be uploaded.
2. Make sure the commit message has been written.
3. Choose to commit directly to the main branch if that option is displayed.
4. Click `Commit changes`.
5. Wait until GitHub finishes saving the changes.

## Step 9: Check the Changes in the Repository

1. Return to the `Code` tab.
2. Check whether the new files have appeared.
3. Open the uploaded files to make sure their content is correct.
4. If there is a new folder, open the folder and check the files inside it.
5. Check whether the README is still displayed properly.

## Step 10: Update the README If Needed

1. If the new files are related to the project, add a description in the README.
2. If an image has been uploaded, display the image in the README.
3. If a demo file such as `index.html` has been uploaded, mention it in the README.
4. Commit the README changes after updating it.

Example of adding an image in README:

```markdown
## Project Display

![Mad_club main display](images/main-display.png)
```

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| File upload failed | Check the internet connection and try uploading again. |
| File is too large | Reduce the file size or use a lighter sample file. |
| File is in the wrong location | Reorganise the file or upload it again into the correct folder. |
| Image does not appear in README | Check the folder name, file name and Markdown image format. |
| Commit message is too general | Use a message that explains the change, such as `Add initial HTML file`. |
