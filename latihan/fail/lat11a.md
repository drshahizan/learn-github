# Exercise 11: Edit README, Commit and Push

## Learning Objective

Participants will be able to add a `readme.md` file in their own GitHub ID folder, record changes using commit and send the changes to GitHub through GitHub Desktop.

## Step 1: Open the Repository in GitHub Desktop

1. Open the GitHub Desktop application.
2. Make sure the active repository is `learn-github`.
3. If another repository is open, select the `learn-github` repository from the repository list.
4. Make sure the GitHub account being used is the participant's own account.
5. Check the `Changes` tab to make sure there are no old unresolved changes.

## Step 2: Open the Repository Folder

1. Click the `Repository` menu in GitHub Desktop.
2. Select `Show in Finder` for macOS or `Show in Explorer` for Windows.
3. The local `learn-github` folder will be displayed.
4. Check the files and folders in the repository.
5. Find the `README.md` file.

## Step 3: Open the Repository Folder in an Editor

1. Open the `learn-github` folder using a suitable editor.
2. The recommended editor is Visual Studio Code.
3. Make sure the entire `learn-github` folder is opened, not only one file.
4. Check the folder structure displayed in the editor.
5. Do not change other files if the facilitator has not requested additional changes.

## Step 4: Open the latihan/id Folder

1. In the editor, find the `latihan` folder.
2. Open the `latihan` folder.
3. Find the `id` folder.
4. Open the `id` folder.
5. The `id` folder will be used to store participant folders based on their own GitHub IDs.

## Step 5: Identify Your Own GitHub ID

1. Open your own GitHub profile.
2. Look at the username in the profile link.
3. Example profile link:

```text
https://github.com/username
```

4. In the example, the GitHub ID is `username`.
5. Use your own GitHub ID to name the exercise folder.

## Step 6: Create Your GitHub ID Folder

1. In the `latihan/id` folder, create one new folder.
2. Name the folder using your GitHub ID.
3. Use the same letters as your GitHub username.
4. Avoid spaces in the folder name.
5. Example folder structure:

```text
latihan/
└── id/
    └── username/
```

## Step 7: Add a readme.md File in Your Folder

1. Open the GitHub ID folder that you created.
2. Create a new file named `readme.md`.
3. Make sure the file name uses lowercase letters.
4. Make sure the file is located inside your GitHub ID folder.
5. Example file location:

```text
latihan/id/username/readme.md
```

6. Add your information notes in the `readme.md` file.
7. Use neat Markdown formatting.
8. Do not include sensitive information.

Example content for `readme.md`:

```markdown
# Participant Information

- Name: Participant Name
- GitHub ID: username
- Institution: Universiti Kebangsaan Malaysia
- Interests: Web development, mobile applications and project documentation

## Exercise Notes

I have successfully cloned the `learn-github` repository to my computer and added a `readme.md` file in my GitHub ID folder.
```

Example final structure:

```text
learn-github/
└── latihan/
    └── id/
        └── username/
            └── readme.md
```

## Step 8: Save the readme.md File

1. Click `Save` in the editor.
2. Make sure the changes are truly saved.
3. Return to GitHub Desktop.
4. Open the `Changes` tab.
5. Make sure the file `latihan/id/username/readme.md` appears in the list of changes.

## Step 9: Check the Changes in GitHub Desktop

1. Click the `readme.md` file in the `Changes` tab.
2. View the displayed changes.
3. Check the text that was added.
4. Make sure the file path is inside your GitHub ID folder.
5. If there is a mistake, return to the editor, fix the file and save it again.

## Step 10: Write a Commit Message

1. Find the `Summary` field in GitHub Desktop.
2. Write a short and clear commit message.
3. The commit message should explain the change made.
4. Avoid messages that are too general, such as `update`, `test` or `edit`.

Example of a good commit message:

```text
Add participant information in GitHub ID folder
```

Another example:

```text
Add GitHub Desktop exercise readme
```

## Step 11: Commit the Changes

1. Make sure the `readme.md` file is selected for commit.
2. Check the commit message.
3. Click the `Commit to main` button.
4. Wait until the commit process is complete.
5. After the commit, the changes have been recorded on the participant's computer.

## Step 12: Push the Changes to GitHub

1. After the commit is complete, find the `Push origin` button.
2. Click `Push origin`.
3. Wait until the push process is complete.
4. Do not close GitHub Desktop while the push process is running.
5. If the push is successful, the changes have been sent to GitHub.

## Step 13: Check the Changes on GitHub Web

1. Open a web browser.
2. Go to the `learn-github` repository on GitHub.
3. Open the `latihan` folder.
4. Open the `id` folder.
5. Open your GitHub ID folder.
6. Open the `readme.md` file.
7. Check whether the added content is displayed.
8. If the changes do not appear, refresh the page or check again whether the push process was successful.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| `readme.md` file does not appear in the Changes tab | Make sure the file has been saved in the editor. |
| Commit button is not active | Make sure there is a changed file and a commit message has been written. |
| Push fails | Check the internet connection and make sure the GitHub account is still active. |
| Changes do not appear on GitHub web | Refresh the page and make sure the push process was successful. |
| File is in the wrong folder | Make sure the file is located at `latihan/id/your-githubid/readme.md`. |
| Other files also changed | Check the changes and do not commit files that are not related to the exercise. |
