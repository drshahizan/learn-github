# Latihan 4: Hasilkan Repositori Projek Pertama

## Objektif Latihan

Peserta dapat mencipta repositori projek pertama dalam GitHub, mengaktifkan fail `README.md` dan menambah maklumat awal supaya repositori mempunyai penerangan yang jelas.

## Langkah 1: Buka Akaun GitHub Sendiri

1. Buka pelayar web.
2. Pergi ke `https://github.com`.
3. Log masuk ke akaun GitHub sendiri.
4. Pastikan peserta berada pada halaman utama GitHub.
5. Klik ikon profil di bahagian atas kanan untuk memastikan akaun yang digunakan adalah betul.

## Langkah 2: Buka Halaman Cipta Repositori Baharu

1. Klik ikon `+` di bahagian atas kanan GitHub.
2. Pilih menu untuk mencipta repositori baharu.
3. GitHub akan memaparkan halaman untuk mencipta repositori baharu.
4. Pastikan nama akaun peserta dipaparkan pada bahagian `Owner`.

## Langkah 3: Namakan Repositori

1. Cari ruangan `Repository name`.
2. Masukkan nama repositori yang sesuai.
3. Gunakan nama yang ringkas, jelas dan mudah difahami.
4. Elakkan nama yang terlalu umum seperti `test`, `latihan1` atau `project`.

Contoh nama repositori projek biasa:

```text
portfolio-saya
projek-web-peribadi
nota-github
mobile-app-demo
data-analytics-mini
```

## Pilihan Repositori Profil GitHub

Repositori `drshahizan/drshahizan` ialah contoh repositori khas kerana nama repositori sama dengan nama pengguna GitHub. Repositori jenis ini boleh digunakan untuk memaparkan README khas pada halaman profil GitHub.

Contoh:

```text
https://github.com/drshahizan/drshahizan
```

Jika nama pengguna peserta ialah `nuraina`, maka repositori profil boleh dinamakan:

```text
nuraina/nuraina
```

Untuk latihan ini, peserta digalakkan mencuba repositori profil jika fasilitator mahu peserta terus membina profil digital.

## Langkah 4: Tulis Description Repositori

1. Cari ruangan `Description`.
2. Tulis penerangan ringkas tentang repositori.
3. Description perlu menjawab soalan tentang tujuan repositori.
4. Gunakan satu ayat yang ringkas dan jelas.

Contoh description:

```text
Repositori profil GitHub untuk portfolio digital.
Repositori portfolio digital pelajar.
Latihan asas GitHub untuk dokumentasi projek.
Projek mini aplikasi web untuk pembelajaran.
Koleksi nota dan latihan Bengkel Asas GitHub.
```

## Langkah 5: Pilih Public atau Private

1. Pilih `Public` jika repositori sesuai dikongsi kepada orang lain.
2. Pilih `Private` jika repositori mengandungi tugasan sensitif, data sulit atau bahan yang belum sesuai dipaparkan.
3. Untuk repositori profil GitHub, pilih `Public` supaya README boleh dipaparkan pada profil.
4. Jika tidak pasti, pilih `Private` dahulu dan bincang dengan fasilitator.

## Langkah 6: Aktifkan README

1. Cari pilihan `Add a README file`.
2. Tandakan pilihan tersebut.
3. README akan menjadi dokumen utama repositori.
4. README membantu orang lain memahami tujuan repositori.
5. Pilihan ini penting kerana peserta akan mengemaskini kandungan `README.md`.

## Langkah 7: Klik Create Repository

1. Semak semula nama repositori.
2. Semak description.
3. Semak pilihan public atau private.
4. Pastikan pilihan `Add a README file` telah ditanda.
5. Klik butang untuk mencipta repositori.

## Langkah 8: Buka Fail README.md

1. Pastikan halaman repositori baharu dipaparkan.
2. Cari fail `README.md` pada senarai fail.
3. Klik fail `README.md`.
4. Klik ikon pensel atau butang edit untuk mengubah kandungan fail.
5. GitHub akan membuka editor untuk fail tersebut.

## Langkah 9: Masukkan Maklumat Awal Dalam README.md

1. Padam kandungan asal jika hanya mengandungi tajuk kosong.
2. Masukkan tajuk utama menggunakan format Markdown.
3. Tambahkan satu perenggan ringkas tentang diri, bidang minat atau tujuan repositori.
4. Gunakan contoh di bawah sebagai rujukan.
5. Ubah nama, peranan, institusi dan minat supaya sesuai dengan diri peserta.

Contoh README untuk rujukan:

```markdown
# Dr Shahizan | Lecturer, Academic Speaker & Tech Educator  <img src="https://raw.githubusercontent.com/drshahizan/drshahizan/master/img/wave.gif" width="30">

Educating future leaders as a Lecturer at [Universiti Teknologi Malaysia (UTM)](https://www.utm.my/), with a strong passion for software development, innovation, and empowering students through dedicated supervision and impactful academic talks.
```

Contoh README untuk peserta:

```markdown
# Nama Peserta | Student, Learner & Future Developer

I am a student at [Universiti Teknologi Malaysia (UTM)](https://www.utm.my/) with an interest in software development, digital portfolio, web technology, data analytics and collaborative project documentation.
```

Contoh README dalam Bahasa Melayu:

```markdown
# Nama Peserta | Pelajar, Pembelajar & Pembangun Masa Hadapan

Saya merupakan pelajar di [Universiti Teknologi Malaysia (UTM)](https://www.utm.my/) yang berminat dalam pembangunan perisian, portfolio digital, teknologi web, analitik data dan dokumentasi projek secara kolaboratif.
```

## Langkah 10: Commit Perubahan README.md

1. Scroll ke bahagian bawah editor.
2. Cari ruangan `Commit changes`.
3. Tulis mesej commit yang jelas.

Contoh mesej commit:

```text
Tambah maklumat awal README profil
```

4. Pilih commit terus ke branch utama jika pilihan tersebut dipaparkan.
5. Klik `Commit changes`.
6. Tunggu sehingga GitHub menyimpan perubahan.

## Langkah 11: Semak Paparan README

1. Kembali ke halaman utama repositori.
2. Semak paparan `README.md`.
3. Pastikan tajuk dipaparkan dengan betul.
4. Pastikan pautan UTM boleh diklik.
5. Jika menggunakan imej `wave.gif`, pastikan imej kecil tersebut muncul.
6. Jika ada kesalahan ejaan, klik edit dan baiki semula.

## Langkah 12: Salin Pautan Repositori

1. Salin pautan repositori daripada address bar.
2. Pautan biasanya berbentuk seperti berikut:

```text
https://github.com/nama-pengguna/nama-repositori
```

3. Simpan pautan tersebut untuk latihan seterusnya.
4. Jika repositori ialah repositori profil, buka profil sendiri dan semak sama ada README dipaparkan pada halaman profil.

## Kebaikan Mengisi README.md Dari Awal

1. Repositori tidak kelihatan kosong.
2. Peserta mula membina identiti digital.
3. Peserta melatih penggunaan Markdown.
4. Fasilitator lebih mudah membuat semakan.
5. Kandungan README boleh dikembangkan menjadi asas portfolio.

## Masalah Biasa dan Cara Mengatasi

| Masalah | Cadangan Penyelesaian |
|---|---|
| README tidak dipaparkan | Pastikan fail dinamakan `README.md` dan berada di root repositori. |
| Imej `wave.gif` tidak muncul | Semak pautan imej dan pastikan sambungan internet berfungsi. |
| Pautan UTM tidak boleh diklik | Pastikan format Markdown betul seperti `[teks](pautan)`. |
| Tersalah ejaan dalam README | Klik ikon pensel, baiki teks dan commit perubahan baharu. |
| README profil tidak muncul pada profil | Pastikan nama repositori sama dengan nama pengguna GitHub dan repositori ialah public. |
