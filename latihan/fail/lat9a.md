# Exercise 9: Install, Sign In and Set Up GitHub Desktop

## Learning Objective

Participants will be able to install GitHub Desktop, sign in using their GitHub account and configure basic settings so that the application can be used for the next exercises.

## Step 1: Open the GitHub Desktop Website

1. Open a web browser.
2. Go to the official GitHub Desktop website:

```text
https://desktop.github.com/
```

3. Make sure the GitHub Desktop website is displayed.
4. Avoid downloading the application from unofficial websites.
5. Click the download button that matches your computer's operating system.

## Step 2: Download GitHub Desktop

1. Click the `Download` button.
2. Wait until the installation file has finished downloading.
3. Check the download location, usually the `Downloads` folder.
4. Make sure the downloaded file is GitHub Desktop.
5. If the download fails, check the internet connection and try again.

## Step 3: Install GitHub Desktop

1. Open the downloaded installation file.
2. Follow the installation instructions shown.
3. Wait until the installation process is complete.
4. Open GitHub Desktop after the installation is complete.
5. If the computer asks for security permission, allow the application to open if it was downloaded from the official GitHub Desktop website.

## Step 4: Sign In to Your GitHub Account

1. On the GitHub Desktop screen, click the option to sign in to GitHub.
2. GitHub Desktop will open a web browser for the authentication process.
3. Enter your GitHub username or email address.
4. Enter your password.
5. Allow GitHub Desktop to access your GitHub account when requested.
6. Return to the GitHub Desktop application after the sign-in process is complete.

## Step 5: Check the Active Account

1. Open GitHub Desktop.
2. Go to the account settings section.
3. Check the GitHub username that is currently active.
4. Make sure it is the account used for the workshop.
5. If the wrong account is used, sign out and sign in again using the correct account.

## Step 6: Set the Commit Name and Email

1. Open the GitHub Desktop settings.
2. Find the section related to `Git` or `Author`.
3. Check the name that will be used for commits.
4. Check the email that will be used for commits.
5. Make sure the name and email match the participant's GitHub account.
6. Update the name and email if needed.

## Step 7: Set the Working Folder

1. Choose a folder location that is easy to find on the computer.
2. Suitable examples:
   - `Documents/GitHub`
   - `Desktop/GitHub`
   - `Documents/GitHub-Training`
3. Avoid choosing a folder that is difficult to find.
4. If using a lab computer, make sure the folder location can be accessed again throughout the workshop.
5. Use the same folder for the repository clone exercise later.

## Step 8: Check That the Application Is Ready to Use

1. Make sure GitHub Desktop can open without problems.
2. Make sure the participant's GitHub account is active.
3. Make sure the commit name and email have been checked.
4. Make sure the working folder has been selected.
5. If everything is complete, participants may continue to Exercise 10.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| Cannot download GitHub Desktop | Check the internet connection and use the official website `https://desktop.github.com/`. |
| Application cannot be opened | Make sure the installation is complete and allow the application through the computer security settings. |
| Cannot sign in | Check the username, email, password and two-factor authentication if enabled. |
| Wrong account is used | Sign out from GitHub Desktop and sign in again using the workshop account. |
| Commit email does not match | Open GitHub Desktop settings and update the commit name and email. |
