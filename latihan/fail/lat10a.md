# Exercise 10: Clone a Repository to the Computer

## Learning Objective

Participants will be able to clone the `drshahizan/learn-github` repository to a computer using GitHub Desktop and check the exercise files locally.

## Step 1: Open the Training Repository

1. Open a web browser.
2. Go to the following link:

```text
https://github.com/drshahizan/learn-github
```

3. Make sure the `drshahizan/learn-github` repository page is displayed.
4. Check the repository owner, which is `drshahizan`.
5. Check the repository name, which is `learn-github`.

## Step 2: Copy the Repository Link

1. On the repository page, click the `Code` button.
2. Make sure the `HTTPS` tab is selected.
3. Copy the repository link displayed.
4. The link used is:

```text
https://github.com/drshahizan/learn-github.git
```

5. Save this link to use in GitHub Desktop.

## Step 3: Open GitHub Desktop

1. Open the GitHub Desktop application.
2. Make sure you have signed in using your own GitHub account.
3. If you have not signed in, complete the sign-in process first.
4. Check the active account in GitHub Desktop.
5. Make sure the application can be used without errors.

## Step 4: Select the Clone Repository Function

1. In GitHub Desktop, click the `File` menu.
2. Select the function to clone a repository.
3. GitHub Desktop will display a window for selecting a repository.
4. If the repository does not appear in your own account list, use the `URL` option.
5. The `URL` option is suitable because the training repository is in the `drshahizan` account.

## Step 5: Enter the Repository Link

1. In the `URL` tab, paste the following link:

```text
https://github.com/drshahizan/learn-github.git
```

2. Check the spelling of the link again.
3. Make sure there are no extra spaces at the beginning or end of the link.
4. GitHub Desktop will display the repository name `learn-github`.
5. If the link is not accepted, copy the link again from GitHub web.

## Step 6: Choose a Local Storage Location

1. Find the `Local path` section.
2. Choose a working folder that is easy to find.
3. Suitable examples:
   - `Documents/GitHub`
   - `Documents/GitHub-Training`
   - `Desktop/GitHub`
4. Make sure you remember the folder location.
5. Avoid saving it in a folder that is difficult to find or in a system folder.

## Step 7: Clone the Repository

1. Click the `Clone` button.
2. Wait until the clone process is complete.
3. The process duration depends on the repository size and internet connection.
4. Do not close GitHub Desktop while the process is running.
5. If successful, the `learn-github` repository will open in GitHub Desktop.

## Step 8: Open the Repository Folder

1. In GitHub Desktop, click the `Repository` menu.
2. Select `Show in Finder` for macOS or `Show in Explorer` for Windows.
3. The `learn-github` folder will open on the computer.
4. Check the files and folders inside it.
5. Find files such as `README.md` or the available exercise folder.

## Step 9: Open the README.md File

1. In the `learn-github` folder, find the `README.md` file.
2. Double-click the file or open it using an editor such as Visual Studio Code.
3. Read the brief content in the README file.
4. Observe the structure of headings, links, images and tables if available.
5. Do not edit the file yet if the facilitator has not asked participants to make changes.

## Step 10: Check the Status in GitHub Desktop

1. Return to GitHub Desktop.
2. Make sure the active repository is `learn-github`.
3. Open the `Changes` tab.
4. If there are no changes yet, the display usually shows that no files have changed.
5. This means the repository has been cloned successfully and is still the same as the original version on GitHub.

## Benefits of Cloning a Repository

1. Participants can view project files locally on the computer.
2. Participants can open files using an editor such as Visual Studio Code.
3. Participants can learn the folder structure of a real project.
4. Participants can make changes, commit and push in the next exercise.
5. Participants understand the relationship between GitHub web, GitHub Desktop and local files.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| Repository link is not accepted | Make sure the link is `https://github.com/drshahizan/learn-github.git`. |
| Clone button cannot be clicked | Check the link and local folder location. |
| Local folder already exists | Choose another location or delete the old folder if it is no longer needed. |
| Clone process takes too long | Check the internet connection and try again. |
| Repository does not appear in your own account | Use the `URL` tab because the repository is in the `drshahizan` account. |
