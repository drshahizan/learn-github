# Exercise 1: Create a GitHub Account

## Learning Objective

Participants will be able to create a new GitHub account using an active email address and ensure that the account is ready for the next workshop activities.

## Step 1: Open the GitHub Website

1. Open a web browser.
2. Type the address `https://github.com`.
3. Press `Enter`.
4. Make sure the GitHub homepage is displayed.
5. Click the `Sign up` button.

## Step 2: Enter Your Email Address

1. In the email field, enter an active email address.
2. Use an email address that you regularly use for learning or professional work.
3. Avoid using temporary email addresses or email accounts that are difficult to access.
4. Click the button to continue the registration process.

## Step 3: Create a Password

1. Enter a strong password.
2. Use a combination of uppercase letters, lowercase letters, numbers and symbols.
3. Avoid using passwords that are too simple, such as your own name, matric number or date of birth.
4. Keep your password in a safe place.
5. Do not share your password with friends during the workshop.

## Step 4: Choose a GitHub Username

1. Enter the username you want to use.
2. Choose a username that is short, professional and easy to read.
3. Avoid usernames that are too casual, difficult to spell or unsuitable for a portfolio.
4. If the username has already been used by someone else, try another variation that still looks professional.

Examples of suitable usernames:

```text
amirulazman
nurainitech
farahdev
hafiz-data
```

## Step 5: Choose Email and Notification Settings

1. GitHub may ask whether you want to receive promotional emails or updates.
2. Select the settings based on your preference.
3. For this workshop, these settings are not critical.
4. Continue with the registration process.

## Step 6: Complete the Security Verification

1. GitHub may display a security challenge.
2. Follow the instructions shown on the screen.
3. Complete the verification until GitHub allows the registration process to continue.
4. If the verification fails, try again carefully.

## Step 7: Verify Your Email Address

1. Open the inbox of the email address used for registration.
2. Look for an email from GitHub.
3. Open the email.
4. Click the verification link or copy the verification code if requested.
5. Return to the GitHub website.
6. Make sure the account has been verified.

## Step 8: Sign In to Your GitHub Account

1. Open `https://github.com` again.
2. Click `Sign in`.
3. Enter your username or email address.
4. Enter your password.
5. Click the sign-in button.
6. Make sure the GitHub dashboard is displayed.

## Step 9: Check That the Account Has Been Created Successfully

1. Click the profile icon at the top right of the page.
2. Select the `Your profile` menu.
3. Make sure the participant's GitHub profile page can be opened.
4. Check the username displayed in the profile link.
5. Copy the GitHub profile link for use in the next exercise.

## Step 10: Record Important Information

Participants should record the following information:

1. GitHub username.
2. GitHub profile link.
3. Email address used.
4. Email verification status.
5. Any issues that occurred during registration.

## Exercise Output

At the end of this exercise, participants should have:

1. An active GitHub account.
2. A verified email address.
3. A suitable GitHub username.
4. A GitHub profile link.
5. The ability to sign in to GitHub again.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| Did not receive the verification email | Check the Spam, Junk or Promotions folder. Make sure the email address entered is correct. |
| Username has already been used | Try adding a field name, a suitable small number or an institutional abbreviation. |
| Forgot password | Use the `Forgot password` function on the sign-in page. |
| Security challenge failed | Read the instructions carefully and try again. |
| Cannot sign in | Check the email address, username and password. Make sure the account has been verified. |

## Security Notes

1. Do not share your password with anyone.
2. Do not use the same password as other important accounts.
3. Use an email address that can still be accessed after the workshop.
4. If you are using a public computer or lab computer, make sure you sign out when you are finished.
