# Latihan Tambahan 2: GitHub Copilot Dalam GitHub Web

Latihan ini membimbing peserta menggunakan **GitHub Copilot melalui GitHub web**. Fokus latihan ialah menggunakan Copilot Chat untuk memahami kandungan repositori, mendapatkan cadangan penambahbaikan README, menjana idea isu, menyemak kod ringkas dan membina amalan penggunaan AI yang bertanggungjawab.

## Objektif Latihan

Selepas melengkapkan latihan ini, peserta dapat:

1. Mengakses GitHub Copilot Chat melalui laman GitHub.
2. Menggunakan Copilot untuk bertanya soalan berkaitan repositori.
3. Mendapatkan cadangan penambahbaikan README.
4. Mendapatkan cadangan isu dan tugasan projek.
5. Menyemak jawapan Copilot sebelum digunakan dalam repositori.

## Langkah 1: Buka GitHub Copilot Di Web

1. Buka pelayar web.
2. Pergi ke pautan berikut:

```text
https://github.com/copilot
```

3. Log masuk menggunakan akaun GitHub anda.
4. Pastikan halaman Copilot Chat dipaparkan.
5. Jika Copilot tidak tersedia, maklumkan kepada fasilitator untuk semakan akses akaun.

## Langkah 2: Mulakan Soalan Pertama

1. Klik ruang pertanyaan pada Copilot Chat.
2. Taip soalan ringkas berkaitan GitHub.
3. Gunakan contoh soalan berikut:

```text
Apakah kegunaan README dalam repositori GitHub?
```

4. Tekan `Enter`.
5. Baca jawapan yang diberikan oleh Copilot.
6. Kenal pasti isi penting yang sesuai digunakan dalam latihan.

## Langkah 3: Buka Repositori Latihan

1. Buka tab baharu dalam pelayar web.
2. Pergi ke repositori latihan anda.
3. Contoh repositori rujukan:

```text
https://github.com/drshahizan/learn-github
```

4. Semak fail dan folder yang terdapat dalam repositori.
5. Buka fail `README.md` jika tersedia.

## Langkah 4: Tanya Copilot Berkaitan Repositori

1. Kembali ke Copilot Chat.
2. Tulis arahan yang jelas berdasarkan repositori latihan.
3. Gunakan contoh arahan berikut:

```text
Berdasarkan sebuah repositori latihan GitHub untuk pelajar, cadangkan bahagian README yang sesuai dimasukkan.
```

4. Baca cadangan Copilot.
5. Pilih bahagian yang sesuai sahaja.
6. Jangan salin semua jawapan tanpa semakan.

## Langkah 5: Dapatkan Cadangan Penambahbaikan README

1. Salin kandungan README projek anda.
2. Tampal kandungan tersebut ke dalam Copilot Chat jika sesuai.
3. Gunakan contoh arahan berikut:

```text
Semak README ini dan cadangkan penambahbaikan dari segi struktur, kejelasan dan maklumat projek. Gunakan Bahasa Melayu yang profesional.
```

4. Semak cadangan yang diberikan.
5. Pilih cadangan yang benar-benar sesuai dengan projek anda.
6. Kemas kini README secara manual dalam GitHub web.

## Langkah 6: Edit README Melalui GitHub Web

1. Buka fail `README.md` dalam repositori anda.
2. Klik ikon pensel untuk mengedit fail.
3. Masukkan penambahbaikan yang telah disemak.
4. Pastikan struktur README mempunyai bahagian penting seperti:

| Bahagian | Kandungan |
|---|---|
| Tajuk projek | Nama projek yang jelas. |
| Ringkasan | Penerangan ringkas tentang tujuan projek. |
| Ciri utama | Fungsi penting projek. |
| Teknologi digunakan | Senarai teknologi atau aplikasi yang digunakan. |
| Ahli kumpulan | Nama ahli yang terlibat. |
| Status projek | Contoh: Dalam pembangunan, selesai atau prototaip. |

5. Semak paparan Markdown sebelum commit.

## Langkah 7: Gunakan Copilot Untuk Cadangan Isu

1. Buka Copilot Chat.
2. Gunakan contoh arahan berikut:

```text
Cadangkan 5 isu yang sesuai untuk projek laman portfolio pelajar di GitHub.
```

3. Pilih satu atau dua isu yang paling sesuai.
4. Buka tab **Issues** dalam repositori.
5. Cipta isu baharu berdasarkan cadangan yang telah disemak.
6. Tulis tajuk isu yang pendek dan jelas.

## Langkah 8: Gunakan Copilot Untuk Semakan Kod Ringkas

1. Buka fail kod ringkas dalam repositori, contohnya `index.html`.
2. Salin bahagian kod yang ingin disemak.
3. Kembali ke Copilot Chat.
4. Gunakan contoh arahan berikut:

```text
Semak kod HTML ini. Cadangkan penambahbaikan ringkas dari segi struktur, kebolehbacaan dan kesesuaian untuk laman portfolio pelajar.
```

5. Baca cadangan Copilot.
6. Gunakan hanya cadangan yang anda faham.
7. Jangan masukkan kod yang tidak difahami ke dalam repositori.

## Langkah 9: Catat Penggunaan AI Dalam README

1. Tambah bahagian ringkas dalam `README.md`.
2. Gunakan tajuk berikut:

```markdown
## Penggunaan AI
```

3. Masukkan catatan seperti contoh berikut:

```markdown
Sebahagian cadangan struktur README dan idea isu projek telah dibantu oleh GitHub Copilot melalui GitHub web. Semua kandungan telah disemak dan disunting semula oleh penulis sebelum digunakan.
```

4. Pastikan catatan ini jelas, jujur dan sesuai dengan keperluan akademik.

## Langkah 10: Commit Perubahan Di GitHub Web

1. Selepas selesai mengedit `README.md`, skrol ke bahagian bawah halaman.
2. Tulis mesej commit yang jelas.
3. Contoh mesej commit:

```text
Tambah penambahbaikan README menggunakan cadangan Copilot
```

4. Pilih pilihan commit terus ke branch yang sedang digunakan.
5. Klik **Commit changes**.
6. Semak semula paparan README selepas commit.

## Amalan Bertanggungjawab

| Perkara | Amalan Yang Disarankan |
|---|---|
| Jawapan Copilot | Semak dahulu sebelum digunakan. |
| Kandungan README | Pastikan maklumat projek adalah benar. |
| Kod | Fahami kod sebelum disimpan dalam repositori. |
| Isu | Pilih isu yang sesuai dengan skop projek. |
| Etika akademik | Nyatakan penggunaan AI jika digunakan dalam tugasan. |
| Data sensitif | Jangan masukkan kata laluan, token, API key atau data peribadi. |

## Masalah Biasa dan Cara Mengatasi

| Masalah | Tindakan |
|---|---|
| Copilot tidak boleh diakses | Semak log masuk GitHub dan status akses Copilot. |
| Jawapan terlalu umum | Berikan konteks projek yang lebih jelas. |
| Cadangan tidak sesuai | Abaikan cadangan tersebut dan tulis semula arahan. |
| README tidak kemas | Gunakan tajuk, senarai bullet dan jadual Markdown. |
| Tidak pasti tentang kod | Tanya fasilitator sebelum commit. |
