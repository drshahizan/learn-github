# Exercise 5: Mad_club Repository

## Learning Objective

Participants will be able to write a `README.md` file for the `Mad_club` repository with a clear, complete structure that is suitable for project documentation.

## Step 1: Open the Mad_club Repository

1. Sign in to your GitHub account.
2. Open the `Mad_club` repository.
3. Make sure you are on the main repository page.
4. Find the `README.md` file.
5. Click the `README.md` file to open it.

## Step 2: Edit the README.md File

1. Click the pencil icon or the `Edit` button.
2. GitHub will open the editor for the `README.md` file.
3. Delete the old text if the content is still empty or incomplete.
4. Prepare the README structure using headings, subheadings and lists.
5. Make sure all project information is written neatly.

## Step 3: Write the Project Title

1. Start the README with a main title.
2. Use the `#` symbol for the main heading in Markdown.
3. The title should be short and describe the project name.

Example:

```markdown
# Mad_club
```

Clearer example:

```markdown
# Mad_club: Club Community and Activity Platform
```

## Step 4: Write the Project Summary

1. Add a `Project Summary` section.
2. Explain the purpose of the project in one or two paragraphs.
3. State the problem the project aims to solve.
4. State the target users of the project.

Example:

```markdown
## Project Summary

Mad_club is a club management system project that helps club members access information about activities, announcements and programmes. This project aims to make communication between club management and members easier.

This system is suitable for student clubs, campus societies or small communities that want to organise activity information more systematically.
```

## Step 5: List the Main Features

1. Add a `Main Features` section.
2. List the important functions proposed for the project.
3. Use bullet points so the list is easy to read.
4. Make sure the listed features are suitable for the `Mad_club` project.

Example:

```markdown
## Main Features

- Club information display
- List of activities and programmes
- Latest announcements
- Committee member information
- Activity photo gallery
- Club social media links
- Member registration form or link
```

## Step 6: State the Technologies Used

1. Add a `Technologies Used` section.
2. List the programming languages, frameworks, libraries or tools used.
3. If the project is still at an early stage, list the planned technologies.
4. Do not include technologies that are not used.

Example:

```markdown
## Technologies Used

- HTML
- CSS
- JavaScript
- Bootstrap
- GitHub Pages
- GitHub for documentation and project management
```

## Step 7: Add Group Members

1. Add a `Group Members` section.
2. List the names of group members.
3. Add each member's role if it has been decided.
4. If roles have not been decided yet, write general roles first and update them later.

Example:

```markdown
## Group Members

| Name | Role |
|---|---|
| Member Name 1 | Project leader |
| Member Name 2 | Frontend developer |
| Member Name 3 | Backend developer |
| Member Name 4 | Documentation and testing |
```

## Step 8: State the Project Status

1. Add a `Project Status` section.
2. State whether the project is still being planned, under development or completed.
3. Use a status that is easy to understand.
4. This status can be updated throughout the project.

Example:

```markdown
## Project Status

Status: Under development
```

## Step 9: Arrange the Complete README

Participants can use the following template for the `Mad_club` repository:

```markdown
# Mad_club: Club Community and Activity Platform

## Project Summary

Mad_club is a club management system project that helps club members access information about activities, announcements and programmes. This project aims to make communication between club management and members easier.

This system is suitable for student clubs, campus societies or small communities that want to organise activity information more systematically.

## Main Features

- Club information display
- List of activities and programmes
- Latest announcements
- Committee member information
- Activity photo gallery
- Club social media links
- Member registration form or link

## Technologies Used

- HTML
- CSS
- JavaScript
- Bootstrap
- GitHub Pages
- GitHub for documentation and project management

## Group Members

| Name | Role |
|---|---|
| Member Name 1 | Project leader |
| Member Name 2 | Frontend developer |
| Member Name 3 | Backend developer |
| Member Name 4 | Documentation and testing |

## Project Status

This project is still in the early development stage. The README structure, main feature list and repository organisation are being updated.
```

## Step 10: Preview the README

1. Click the `Preview` tab if available.
2. Check the project title display.
3. Check the group members table.
4. Make sure the bullet list is displayed correctly.
5. Fix spelling, formatting or arrangement if needed.

## Step 11: Commit the Changes

1. Scroll to the bottom of the editor.
2. Find the `Commit changes` section.
3. Write a clear commit message.

Example commit message:

```text
Add Mad_club project README
```

4. Click `Commit changes`.
5. Wait until the changes are saved.

## Step 12: Check the Final README Display

1. Return to the main page of the `Mad_club` repository.
2. Check the README display below the file list.
3. Make sure all main sections are present.
4. If there are mistakes, click edit and update it again.

## Benefits of a Complete README

1. Makes it easier for others to understand the project.
2. Helps facilitators review the work.
3. Shows documentation maturity.
4. Becomes the foundation of a digital portfolio.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| README is not displayed | Make sure the file is named `README.md` and is located at the repository root. |
| Table does not display correctly | Make sure the `|` symbols and separator line `|---|---|` are written correctly. |
| Heading is not large | Make sure the heading starts with `#` at the beginning of the line. |
| List is not neat | Use `-` for each list item. |
| Forgot to commit | Scroll to the bottom and click `Commit changes` after editing. |
