# Additional Exercise 3: Connect VS Code with GitHub

This exercise guides participants in connecting **Microsoft Visual Studio Code** with a **GitHub** account. The focus is on signing in to GitHub in VS Code, opening the training repository, making file changes and sending the changes back to GitHub through the graphical interface.

## Learning Objectives

After completing this exercise, participants will be able to:

1. Sign in to a GitHub account through VS Code.
2. Connect VS Code with a GitHub account.
3. Clone the training repository to a computer.
4. Edit files in VS Code.
5. Commit and push changes to GitHub.

## Step 1: Open VS Code

1. Open the **Visual Studio Code** application on the computer.
2. Make sure the computer has an internet connection.
3. Make sure you already have an active GitHub account.
4. If VS Code is not installed, inform the facilitator.

## Step 2: Sign In to a GitHub Account

1. Click the **Accounts** icon at the bottom left of VS Code.
2. Select **Sign in to GitHub**.
3. A web browser will open automatically.
4. Sign in using your GitHub account.
5. Click **Authorize Visual Studio Code** if requested.
6. Return to VS Code after the sign-in process is complete.

## Step 3: Check the Sign-In Status

1. Click the **Accounts** icon again.
2. Make sure your GitHub account name is displayed.
3. If the account is not displayed, repeat the sign-in process.
4. Make sure there are no error messages related to account connection.

## Step 4: Copy the Training Repository Link

1. Open a web browser.
2. Go to the following training repository:

```text
https://github.com/drshahizan/learn-github
```

3. Click the **Code** button.
4. Select the **HTTPS** tab.
5. Copy the repository link displayed.

## Step 5: Clone the Repository in VS Code

1. Return to VS Code.
2. Click the **Source Control** icon on the left panel.
3. Click **Clone Repository**.
4. Paste the repository link that you copied.
5. Choose a folder location on the computer to save the training files.
6. Wait until the clone process is complete.
7. Click **Open** when VS Code asks to open the repository folder.

## Step 6: Check the Repository Files

1. View the **Explorer** panel on the left.
2. Make sure the repository files and folders are displayed.
3. Open the `README.md` file.
4. Read the content of the file.
5. Make sure you are in the correct repository folder.

## Step 7: Add an Exercise Folder and File

1. In the **Explorer** panel, find the `latihan` folder.
2. If the `latihan` folder does not exist, create a new folder named `latihan`.
3. Inside the `latihan` folder, create a new folder named `id`.
4. Inside the `id` folder, create a new folder using your GitHub ID.
5. Example folder name:

```text
latihan/id/your-githubid
```

6. Inside your GitHub ID folder, create a new file named `readme.md`.
7. Enter brief information about yourself in the file.
8. Example content:

```markdown
# Brief Profile

Name: Your Name
GitHub ID: your-githubid
Area of Interest: Web development, mobile applications or data
Workshop Purpose: Learn to use GitHub for portfolios and project collaboration
```

9. Save the file after writing the information.

## Step 8: Check Changes in Source Control

1. Click the **Source Control** icon on the left panel of VS Code.
2. Check the list of changed files.
3. Make sure the `readme.md` file in your GitHub ID folder is displayed.
4. If the file is not displayed, make sure the file has been saved.

## Step 9: Write a Commit Message

1. In the commit message field, write a clear message.
2. Example commit message:

```text
Add GitHub ID training profile
```

3. Make sure the commit message explains the change made.
4. Avoid messages that are too general, such as `update` or `test`.

## Step 10: Commit the Changes

1. Click the **Commit** button.
2. If VS Code asks for confirmation, choose the appropriate option to continue the commit.
3. Check that the changes have been recorded.
4. If there is an error, read the error message and inform the facilitator.

## Step 11: Push the Changes to GitHub

1. Click **Sync Changes** or **Push** in VS Code.
2. Wait until the process is complete.
3. If asked to sign in again, follow the instructions displayed.
4. Make sure there are no error messages after the push process.

## Step 12: Check on GitHub Web

1. Open the training repository through a web browser.
2. Check the `latihan/id` folder.
3. Find your GitHub ID folder.
4. Open the `readme.md` file.
5. Make sure the information you added is displayed correctly.
6. Copy the file link if requested by the facilitator.

## Common Problems and How to Solve Them

| Problem | Action |
|---|---|
| Cannot sign in to GitHub | Check the email, password and internet connection. |
| Clone button does not appear | Open the Source Control tab or use the Command Palette. |
| File does not appear in Source Control | Make sure the file has been saved in the repository folder. |
| Push fails | Check GitHub sign-in and make sure you have access to the repository. |
| Folder is in the wrong location | Check the folder structure again before committing. |
