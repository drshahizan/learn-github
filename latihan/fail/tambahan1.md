# Latihan Tambahan 1: Bina dan Terbitkan Laman github.io

## Contoh Rujukan

Sebelum peserta membina laman sendiri, peserta boleh melihat contoh laman `github.io` melalui pautan berikut:

```text
https://drshahizan.github.io/
```

Laman `https://drshahizan.github.io/` ialah contoh laman web yang diterbitkan menggunakan GitHub Pages. Pautan seperti ini biasanya menggunakan format:

```text
https://nama-pengguna.github.io/
```

Dalam contoh ini, `drshahizan` ialah nama pengguna GitHub, manakala `github.io` ialah domain yang digunakan oleh GitHub Pages untuk menerbitkan laman web.

Laman seperti ini boleh digunakan sebagai portfolio digital, laman profil akademik, laman projek, dokumentasi bengkel atau pusat pautan kepada bahan pembelajaran.

## Apa Itu github.io

`github.io` ialah alamat laman web yang disediakan melalui GitHub Pages. Apabila pengguna GitHub mencipta repositori khas dengan nama:

```text
nama-pengguna.github.io
```

GitHub boleh menerbitkan kandungan dalam repositori tersebut sebagai laman web.

Contoh:

| Nama Pengguna GitHub | Nama Repositori | Pautan Laman |
|---|---|---|
| `drshahizan` | `drshahizan.github.io` | `https://drshahizan.github.io/` |
| `nuraina` | `nuraina.github.io` | `https://nuraina.github.io/` |
| `amiruldev` | `amiruldev.github.io` | `https://amiruldev.github.io/` |

Jika peserta mempunyai nama pengguna GitHub `ali-dev`, maka repositori untuk laman `github.io` boleh dinamakan:

```text
ali-dev.github.io
```

Pautan laman yang terhasil ialah:

```text
https://ali-dev.github.io/
```

## Kegunaan Laman github.io

1. Portfolio digital untuk memaparkan maklumat diri, kemahiran dan projek pilihan.
2. Demo projek web statik seperti HTML, CSS dan JavaScript.
3. Dokumentasi pembelajaran seperti nota, latihan, bahan bengkel atau panduan projek.
4. Profil akademik atau profesional.
5. Pusat pautan kepada GitHub, LinkedIn, Google Scholar, portfolio, projek atau bahan kursus.

## Perbezaan Profil GitHub dan Laman github.io

| Perkara | Profil GitHub | Laman github.io |
|---|---|---|
| Alamat | `https://github.com/nama-pengguna` | `https://nama-pengguna.github.io/` |
| Fungsi utama | Paparan akaun, repositori dan aktiviti GitHub | Laman web bebas untuk portfolio atau demo |
| Kandungan | Dikawal oleh paparan GitHub | Direka sendiri menggunakan HTML, CSS atau framework statik |
| Sesuai untuk | Menunjukkan repositori dan aktiviti | Menunjukkan portfolio, profil, projek dan pautan penting |
| Contoh | `https://github.com/drshahizan` | `https://drshahizan.github.io/` |

## Objektif Latihan

Peserta dapat membina laman `github.io` ringkas menggunakan GitHub Pages dan menerbitkan halaman portfolio asas yang boleh dibuka melalui pelayar web.

## Langkah 1: Semak Nama Pengguna GitHub

1. Log masuk ke akaun GitHub.
2. Klik ikon profil di bahagian atas kanan.
3. Pilih `Your profile`.
4. Lihat pautan profil GitHub.
5. Nama selepas `github.com/` ialah nama pengguna GitHub.

Contoh:

```text
https://github.com/nuraina
```

Nama pengguna ialah:

```text
nuraina
```

## Langkah 2: Cipta Repositori github.io

1. Klik ikon `+` di bahagian atas kanan GitHub.
2. Pilih menu untuk mencipta repositori baharu.
3. Pada ruangan nama repositori, taip nama dalam format berikut:

```text
nama-pengguna.github.io
```

4. Jika nama pengguna ialah `nuraina`, nama repositori ialah:

```text
nuraina.github.io
```

5. Pastikan ejaan nama pengguna sama seperti akaun GitHub sendiri.
6. Pilih `Public`.
7. Aktifkan `Add a README file`.
8. Klik butang untuk mencipta repositori.

## Langkah 3: Tambah Fail index.html

1. Buka repositori `nama-pengguna.github.io`.
2. Klik `Add file`.
3. Pilih `Create new file`.
4. Namakan fail:

```text
index.html
```

5. Masukkan kod HTML ringkas.

Contoh:

```html
<!DOCTYPE html>
<html lang="ms">
<head>
  <meta charset="UTF-8">
  <title>Portfolio Saya</title>
</head>
<body>
  <h1>Nama Peserta</h1>
  <p>Selamat datang ke laman portfolio GitHub Pages saya.</p>

  <h2>Tentang Saya</h2>
  <p>Saya berminat dalam pembangunan web, dokumentasi projek dan teknologi digital.</p>

  <h2>Pautan</h2>
  <ul>
    <li><a href="https://github.com/nama-pengguna">Profil GitHub</a></li>
    <li><a href="https://www.linkedin.com/">LinkedIn</a></li>
  </ul>
</body>
</html>
```

6. Ubah `Nama Peserta` kepada nama sendiri.
7. Ubah `nama-pengguna` kepada GitHub ID sendiri.

## Langkah 4: Commit Fail index.html

1. Scroll ke bahagian bawah halaman.
2. Cari bahagian `Commit changes`.
3. Tulis mesej commit yang jelas.

Contoh:

```text
Tambah laman portfolio asas
```

4. Klik `Commit changes`.
5. Pastikan fail `index.html` wujud dalam repositori.

## Langkah 5: Aktifkan GitHub Pages

1. Buka tab `Settings` dalam repositori.
2. Cari bahagian `Pages`.
3. Pada bahagian sumber, pilih branch utama seperti `main`.
4. Pilih folder root jika diminta.
5. Klik `Save`.
6. Tunggu beberapa minit sehingga GitHub menjana pautan laman.

## Langkah 6: Buka Pautan github.io

1. Buka pautan laman dalam format:

```text
https://nama-pengguna.github.io/
```

2. Contoh:

```text
https://nuraina.github.io/
```

3. Semak sama ada laman berjaya dipaparkan.
4. Jika belum muncul, tunggu beberapa minit dan refresh semula.
5. Salin pautan laman untuk dimasukkan ke profil GitHub atau README projek.

## Langkah 7: Kemas Kini Profil GitHub

1. Buka profil GitHub sendiri.
2. Klik `Edit profile`.
3. Masukkan pautan `github.io` pada ruangan website atau pautan profil.
4. Simpan perubahan.
5. Semak semula profil GitHub.

## Kebaikan Laman github.io

1. Peserta mempunyai laman portfolio ringkas.
2. Projek web statik boleh dipaparkan kepada orang lain.
3. Pautan mudah dikongsi kepada pensyarah, komuniti atau majikan.
4. Peserta belajar hubungan antara repositori dan laman web.
5. Laman boleh dikembangkan kemudian dengan CSS, JavaScript atau framework statik.

## Masalah Biasa dan Cara Mengatasi

| Masalah | Cadangan Penyelesaian |
|---|---|
| Laman tidak muncul | Tunggu beberapa minit dan refresh semula. |
| Pautan salah | Pastikan format `https://nama-pengguna.github.io/` betul. |
| Repositori salah nama | Pastikan nama repositori sama dengan `nama-pengguna.github.io`. |
| Halaman kosong | Pastikan fail `index.html` wujud di root repositori. |
| Perubahan tidak muncul | Semak commit terakhir dan tunggu GitHub Pages selesai diterbitkan. |
