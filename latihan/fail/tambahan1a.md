# Additional Exercise 1: GitHub Pages

## Reference Example

Before participants build their own website, they can view an example of a `github.io` website through the following link:

```text
https://drshahizan.github.io/
```

The website `https://drshahizan.github.io/` is an example of a website published using GitHub Pages. Links like this usually use the following format:

```text
https://username.github.io/
```

In this example, `drshahizan` is the GitHub username, while `github.io` is the domain used by GitHub Pages to publish websites.

Websites like this can be used as digital portfolios, academic profile pages, project websites, workshop documentation or link hubs for learning materials.

## What Is github.io

`github.io` is a website address provided through GitHub Pages. When a GitHub user creates a special repository with the name:

```text
username.github.io
```

GitHub can publish the content in that repository as a website.

Example:

| GitHub Username | Repository Name | Website Link |
|---|---|---|
| `drshahizan` | `drshahizan.github.io` | `https://drshahizan.github.io/` |
| `nuraina` | `nuraina.github.io` | `https://nuraina.github.io/` |
| `amiruldev` | `amiruldev.github.io` | `https://amiruldev.github.io/` |

If a participant has the GitHub username `ali-dev`, the repository for the `github.io` website can be named:

```text
ali-dev.github.io
```

The resulting website link is:

```text
https://ali-dev.github.io/
```

## Uses of a github.io Website

1. A digital portfolio to display personal information, skills and selected projects.
2. A demo for static web projects such as HTML, CSS and JavaScript.
3. Learning documentation such as notes, exercises, workshop materials or project guides.
4. An academic or professional profile.
5. A link hub for GitHub, LinkedIn, Google Scholar, portfolios, projects or course materials.

## Difference Between a GitHub Profile and a github.io Website

| Item | GitHub Profile | github.io Website |
|---|---|---|
| Address | `https://github.com/username` | `https://username.github.io/` |
| Main function | Displays GitHub account, repositories and activities | Independent website for a portfolio or demo |
| Content | Controlled by GitHub's display | Designed by the user using HTML, CSS or a static framework |
| Suitable for | Showing repositories and activities | Showing portfolios, profiles, projects and important links |
| Example | `https://github.com/drshahizan` | `https://drshahizan.github.io/` |

## Learning Objective

Participants will be able to build a simple `github.io` website using GitHub Pages and publish a basic portfolio page that can be opened through a web browser.

## Step 1: Check Your GitHub Username

1. Sign in to your GitHub account.
2. Click the profile icon at the top right.
3. Select `Your profile`.
4. View your GitHub profile link.
5. The name after `github.com/` is your GitHub username.

Example:

```text
https://github.com/nuraina
```

The username is:

```text
nuraina
```

## Step 2: Create a github.io Repository

1. Click the `+` icon at the top right of GitHub.
2. Select the menu to create a new repository.
3. In the repository name field, type the name using the following format:

```text
username.github.io
```

4. If the username is `nuraina`, the repository name is:

```text
nuraina.github.io
```

5. Make sure the username spelling is the same as your own GitHub account.
6. Select `Public`.
7. Enable `Add a README file`.
8. Click the button to create the repository.

## Step 3: Add an index.html File

1. Open the `username.github.io` repository.
2. Click `Add file`.
3. Select `Create new file`.
4. Name the file:

```text
index.html
```

5. Enter simple HTML code.

Example:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My Portfolio</title>
</head>
<body>
  <h1>Participant Name</h1>
  <p>Welcome to my GitHub Pages portfolio website.</p>

  <h2>About Me</h2>
  <p>I am interested in web development, project documentation and digital technology.</p>

  <h2>Links</h2>
  <ul>
    <li><a href="https://github.com/username">GitHub Profile</a></li>
    <li><a href="https://www.linkedin.com/">LinkedIn</a></li>
  </ul>
</body>
</html>
```

6. Change `Participant Name` to your own name.
7. Change `username` to your own GitHub ID.

## Step 4: Commit the index.html File

1. Scroll to the bottom of the page.
2. Find the `Commit changes` section.
3. Write a clear commit message.

Example:

```text
Add basic portfolio website
```

4. Click `Commit changes`.
5. Make sure the `index.html` file exists in the repository.

## Step 5: Enable GitHub Pages

1. Open the `Settings` tab in the repository.
2. Find the `Pages` section.
3. In the source section, select the main branch such as `main`.
4. Select the root folder if requested.
5. Click `Save`.
6. Wait a few minutes for GitHub to generate the website link.

## Step 6: Open the github.io Link

1. Open the website link in this format:

```text
https://username.github.io/
```

2. Example:

```text
https://nuraina.github.io/
```

3. Check whether the website is displayed successfully.
4. If it has not appeared yet, wait a few minutes and refresh again.
5. Copy the website link to add it to your GitHub profile or project README.

## Step 7: Update Your GitHub Profile

1. Open your own GitHub profile.
2. Click `Edit profile`.
3. Enter the `github.io` link in the website or profile link field.
4. Save the changes.
5. Check your GitHub profile again.

## Benefits of a github.io Website

1. Participants have a simple portfolio website.
2. Static web projects can be displayed to others.
3. The link is easy to share with lecturers, communities or employers.
4. Participants learn the relationship between repositories and websites.
5. The website can be expanded later with CSS, JavaScript or a static framework.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| Website does not appear | Wait a few minutes and refresh again. |
| Link is wrong | Make sure the format `https://username.github.io/` is correct. |
| Repository name is wrong | Make sure the repository name matches `username.github.io`. |
| Page is blank | Make sure the `index.html` file exists at the repository root. |
| Changes do not appear | Check the latest commit and wait for GitHub Pages to finish publishing. |
