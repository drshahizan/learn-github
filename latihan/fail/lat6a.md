# Exercise 6: Using Markdown in a README File

## Learning Objective

Participants will be able to use basic Markdown formatting in a `README.md` file so that project documentation is more organised, easier to read and more professional.

## Step 1: Open the README.md File

1. Sign in to your GitHub account.
2. Open a project repository, for example `Mad_club`.
3. Find the `README.md` file.
4. Click the file to view the existing content.
5. Click the pencil icon or the `Edit` button to update the content.

## Step 2: Add Headings and Subheadings

1. Use the `#` symbol for the main heading.
2. Use `##` for main subheadings.
3. Use `###` for smaller subheadings.
4. Make sure there is a space after the `#` symbol.

Example:

```markdown
# Mad_club: Club Community and Activity Platform

## Project Summary

### Project Objectives
```

## Step 3: Add a Bullet List

1. Use the `-` symbol for bullet lists.
2. Each item should be written on a new line.
3. Use bullet lists for main features, technologies or tasks.

Example:

```markdown
## Main Features

- Club information display
- List of activities and programmes
- Latest announcements
- Activity photo gallery
- Club social media links
```

## Step 4: Add a Numbered List

1. Use a number followed by a full stop.
2. Numbered lists are suitable for usage steps or work processes.
3. Make sure the numbers are arranged in sequence.

Example:

```markdown
## How to Use

1. Open the project homepage.
2. Select the activity menu.
3. Check the latest announcements.
4. Fill in the registration form if interested.
```

## Step 5: Add Links

1. Use the format `[link text](link address)`.
2. The link text should describe the destination.
3. Make sure the link address is complete and correct.
4. Test the link after the README is saved.

Example:

```markdown
[Universiti Teknologi Malaysia Website](https://www.utm.my/)
[Mad_club Project Demo](https://username.github.io/Mad_club/)
```

## Step 6: Add Images

1. Prepare an image such as a project screenshot.
2. Upload the image to an `images` or `assets` folder in the repository.
3. Use the format `![alternative text](image-location)`.
4. The alternative text should briefly describe the image.

Example:

```markdown
## Project Display

![Mad_club main display](images/main-display.png)
```

## Step 7: Add a Simple Table

1. Use the `|` symbol to create a table.
2. The first row is the column header.
3. The second row is the column separator.
4. The following rows contain the table content.

Example:

```markdown
## Group Members

| Name | Role |
|---|---|
| Member Name 1 | Project leader |
| Member Name 2 | Frontend developer |
| Member Name 3 | Documentation and testing |
```

## Step 8: Add Simple Code

1. Use single backticks for short code in one line.
2. Use three backticks for a code block.
3. Short code is suitable for file names, short commands or structure examples.
4. Do not include passwords, tokens or API keys in code blocks.

Example of code in a sentence:

```markdown
The main project file is `index.html`.
```

Example of a code block:

````markdown
```html
<h1>Welcome to Mad_club</h1>
<p>Club community and activity platform.</p>
```
````

## Step 9: Preview the README

1. Click the `Preview` tab.
2. Check the heading and subheading display.
3. Check the bullet and numbered lists.
4. Check the links and images.
5. Check the simple table.
6. Check the code display.
7. Fix the format if any section is not displayed correctly.

## Step 10: Commit the Changes

1. Scroll to the bottom of the editor.
2. Find the `Commit changes` section.
3. Write a clear commit message.

Example commit message:

```text
Add Markdown formatting to README
```

4. Click `Commit changes`.
5. Open the main repository page again to view the final display.

## Common Problems and How to Solve Them

| Problem | Suggested Solution |
|---|---|
| Heading is not displayed large | Make sure there is a space after the `#` symbol. Example: `# Title`, not `#Title`. |
| List does not display correctly | Make sure each item is on a new line and starts with `-` or a number. |
| Link cannot be clicked | Check the format `[text](link)` and make sure the link is complete. |
| Image does not appear | Check the file name, folder location and format `![text](image-location)`. |
| Table is not neat | Make sure the separator row `|---|---|` exists and the number of columns matches. |
| Code block is not closed | Make sure the opening and closing three backticks are written correctly. |
